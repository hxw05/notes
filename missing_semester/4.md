# 4. Data Wrangling

Data Wrangling简单而言， 就是对数据进行的各种变换，例如对txt文件内容的可视化，对数据的格式转换。

## 例子：通过`journalctl`查看远程服务器上的尝试登录记录，并提取出其中的用户名

- ssh host command 在远端执行指令，其中command需要注意分割的问题，这将影响该指令是在本地还是在远端运行。

尝试从journalctl中获取其中含有指定关键字的行

- grep
- less *the pager*

```bash
ssh tsp 'journalctl | grep ssh | grep "Disconnected from"' | less
```

## `sed` - the stream editor

sed is a modification of a much earlier program called “ed” which was a really weird editor.

删除 “Disconnected from”之前的无用信息

```bash
!! | sed 's/.*Disconnected from//'
## 's' stands for 'substitute'
```

此处格式：`s/search string/replacement string/`

在这里的正则表达式同样默认只会执行一次（针对第一个匹配）

`s/search string/replacement string/g` ← the g modifier 将正则表达式转换为全局

`s/(ab)*//g`  直接传入sed无法达到想要的效果，由于历史原因，需要添加-E flag来启用一些现代化的语法。如果不添加 **-E** flag，在这个例子下需要对()进行转义来**让它们具有特殊含义**，否则默认只会匹配literal parenthesis

`s/.*Disconnected from//`  注意其Greedy性需要用?来进行限制

*anchor the pattern* - 为正则表达式添加开始和结束记号^$，提升鲁棒性。这样做，避免了在pattern中要匹配的子串恰好满足整个pattern的要求的特殊状况（*oddity）*。

## `sort`, `uniq`

sort可将输入的数据进行排序

```bash
!! | sort
```

uniq可对输入的数据去重，也可以统计输入中重复内容的数量

```bash
!! | uniq -c
```

现在，如果需要统计日志中出现了最多次的用户名，在应用sed指令以后，进行如下wrangling

```bash
!! | sort | uniq -c | sort -n -k1,1 | tail -n10
## sort -n 按数字进行排序
## sort -k1,1 按输入的第一列开始排序、第一列结束排序
```

默认按照升序排列，用tail获取最后几行就可以得到使用次数相对最多的用户名。

在这里-k不是必要的，sort默认会根据每一行的内容进行排序，而uniq -c输出的内容中，数字刚好在开头的位置，因此会被直接当作依据。

## `awk` - a column based stream processor

more focused on columnar data

awk默认会以空格分隔的模式解析它的输入

```bash
!! | awk '{print $2}'
## 只要第二列的内容
```

very powerful language for this kind of data wrangling

*really* fancy things

回到这一步

```bash
!! | sort | uniq -c
```

假设我们在尝试登录的用户名中获取

- 只出现过一次（只尝试登录过一次）
- 以c开头，以e结尾

使用awk来实现

```bash
!! | awk '$1 == 1 && $2 ~ /^c.*e$/ {print $0}'
```

显然，awk后的输入中，`$1 == 1 && $2 ~ /^c.*e$/`  对行进行了筛选，且筛选条件里的单位是列的元素。

在awk中，`BEGIN`和`END`分别是两个特殊的筛选条件，它们匹配首行和尾行。可用awk来实现类似于wc -l的计数

```bash
!! | awk 'BEGIN {rows = 0} $1 == 1 && $2 ~ /^c.*e$/ {rows += 1} END {print $0}'
```

首行将rows置0，之后的每一个匹配行都令rows + 1，最后将结果输出来。

这样做的意义在于，可以通过awk来实现一些定制化的操作，例如这里就对计数进行了细化（只不过可能使用场景都比较特殊）。

## `paste`

`-s`将多行数据转换为单行，`-d ,`设定了分隔符为逗号

```bash
!! | paste -sd,
```

## `bc`

*Berkley Calculator?*

一个命令行内的计算器，从标准输入读取数据

```bash
echo "1+2" | bc -l
```

**why `-l`?** - because many of these programs normally operate in a stupid mode where they’re unhelpful. (39:03)

用bc来计算所有登录尝试超过一次的用户名的总使用次数

```bash
!! | awk '$1 != 1 {print $1}' | paste -sd+ | bc -l
```
