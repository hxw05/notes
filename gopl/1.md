# Ch1. Tutorial

## package, import
- main包是一个特殊的包，它定义了一个程序而不是一个库。这里对于库与可执行程序的区分非常像rust中的lib crate/非lib crate。main函数是一个特殊的函数，定义了程序的起点。
- 多余的import会导致编译失败，这个严格的规则避免了随着程序的迭代导致无用包增加的风险。
- **import语句必须紧跟在package之后。**

## 末尾不带分号
Go语句的结尾不需要带分号，实际上是换行符+某些token的组合会被自动看作是类似于分号的角色。这也说明在Go中对代码**正确换行**也很重要。
- 左花括号不能另起一行，必须跟在func、if、for、...之后
- x+y等表达式中，+之后才能换行，之前不可换行

## gofmt
Go很看重代码的格式化，可以使用`gofmt`命令行工具进行格式化，或者用`go fmt`指令对项目中的文件进行格式化。
编辑器可以在保存时自动帮你运行gofmt。

## goimports
goimports用于管理包的导入和导出，它不是一个自带的工具，需要单独下载。

## 自增/自减运算符
Go中的自增自减运算符只能放在操作数的后面，并且自增自减不属于表达式，而是语句，因此不能赋值给变量。

## 空标识符
`_`是Go中的空标识符，用于当语法要求这个地方需要一个变量来接收，但是业务逻辑并不需要的时候。

## 四种定义变量的方式
```go
s := ""
var s string
var s = ""
var s string = ""
```
第一种简洁定义的写法只能用来定义局部变量（eg. 在函数内部）。
一般使用的是前两种，其中第一种写法表示初始值有意义，第二种写法初始值为零值，表示初始值不重要或者采用零值。

## map
map是一种引用类型。它的key的类型必须是一个comparable，它的值可以是任意类型。
当访问一个不存在的key时，将会返回它的零值。例如在使用哈希表统计频率的过程中：
```go
counts[input.Text()]++
```
这个自增语句是**等价于其+=的写法**的，所以`counts[input.Text]`出现在了右侧，将计算为它的零值，也就是0。
遍历map的时候，其顺序在标准中未给出，但是在实际实现中是随机的，且这是一个刻意的设计，使得程序不依赖于map中键值对的顺序。

## struct，组合字面量
> A struct is a group of values called *fields*. (p15)

对于struct的定义有一种写法叫做**组合字面量（composite literals）**，是指类似于`[]color.Color{...}`和`gif.GIF{}`这样的写法。这种写法可以一步到位地定义一个struct，而不是先将零值声明出来再一个个单独赋值。map也有组合字面量的写法。

## channel的发送与接收

当发送或接收channel时都会导致goroutine的阻塞，直到其发送的内容被接收/接收的内容被发送，也就是值正确地传递了。

## switch语句

Go中的switch语句有下面的特点：
- 默认不会穿透，除非给定`fallthrough`关键字
- 不需要break
- switch不一定要跟上一个操作数，不跟的情况下相当于 `switch true`，具体看分支case是否返回true

## 指针

C中的指针相对来说没有什么限制，可以对它做很多操作；许多语言（例如JS）中与指针沾边的类型则被暴露为一种引用，除了可以传来传去之外并不能做什么操作。Go中的指针介于二者之间，可以使用`&`来取地址，`*`来解引用，但不能做算术运算。
