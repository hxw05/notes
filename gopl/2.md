# Ch2. Project Structure

## 关键字、预声明名称

Go有25个关键字。关键字是被保留的，不能用作标识符。除了关键字以外，还有一类**预定义名称（predeclared names）**，例如int、error、true等。这些名称都不是保留的，可以用作标识符，但是为了避免混淆，只有一少部分在少数情况下才被使用。

## 命名

一般来说倾向于比较短的简洁名称，例如使用`i`好过使用`theLoopIndex`。不过这是对于局部变量而言。变量存在的范围越大，其名称包含的信息量就应该更多以便区分。

命名通常使用camelCase（或者PascalCase，当需要导出一个标识符时）。对于camelCase中出现的固有缩写词原样使用，例如`escapeHTML`而不是~~`escapeHtml`~~，`getElementByID`而不是`getElementById`（不过由于id只有两个字母，区别不大），以及`NaN`不需要刻意写成`nAN`（camelCase）或者`NAN`（PascalCase）。

## 声明（Declaration）

Go中有四种声明：var、const、func、type。

## 变量

变量的声明方式是 `var name type = expression`，`type`和`= expression`可以择其一省略，也是一个合法的声明。如果省略后者，那么变量将会被赋予一个零值，这个零值是该类型下的一个合理存在的值。于是Go中不存在任何一个“未定义的值”，Go中的声明和定义是一体的。

由于零值的存在，通常我们会愿意去赋予零值一个合理的意义用于实际的业务逻辑（例如默认值），这样一个变量一旦声明出来，它的值就具有意义，而不是停滞在没有意义的状态直到被定义（赋值）。常见的“零值即可用”的结构包括`sync.Mutex`/`sync.RWMutex`、`bytes.Buffer`等。

var语句不一定要声明相同类型的变量。当没有给出变量的类型时可以使用不同类型的变量。但是最好只有在这些变量拼在一起有实际含义的时候采用这种紧凑的写法。

```go
var b, f, s = true, 2.3, "four"
```

## 短变量声明

短变量声明用`:=`进行。局部变量可以使用且经常使用这种方式声明。短变量声明可以多个一起，其条件是至少有一个变量被声明出来。如果位于左侧的某些变量已经存在，则其效果与赋值相同。

## 指针

变量是用来容纳一个值的标识符，又可以称为是一个可取地址的值。一个值不一定有名字，例如结构体的字段，它并不是一个变量而是依附于结构体变量上的一个值。像这些没有名字的值就可以用指针来指代和间接（indirect）访问。

如果一个函数返回一个变量的指针，那么这个变量不会在函数返回后被释放，因为可达性仍然维持。

### `flag` package

flag是一个用来解析命令行参数的包。通过将变量赋值为`flag.Bool`、`flag.String`等函数的返回值来使它们接收来自命令行的参数。它们的返回值都是指针。

在main函数中，需要先调用`flag.Parse()`对传入程序的参数进行解析。解析以后，指针变量指向的值会被改变，这样指针变量通过解引用就可以拿到解析后的值。另外，如果`flag.Parse()`在解析过程中出现问题会`os.Exit(2)`。

### `new` 函数

new是一个预定义的名称，它是一个内置的函数（predeclared function）。一般用不到new的时候可以直接把变量起名叫new也没有问题。

new的作用是通过`new(T)`构造一个没有名称的值（也就是它没有对应的变量存储），返回这个值的地址。它可以用在我们不需要为变量起名字的情况下。需要注意new背后没有什么特殊的操作，它只是一个比较简便的语法（syntactic convenience）。

下面两种写法是等价的

```go
func newInt() *int {
    return new(int)
}
```

```go
func newInt() *int {
    var a int
    return &a // 在这里就是前面提到的其实根本不需要为这个变量起名的情形
}
```

## 变量的生命周期

在Go中，GC的方法与JS类似，即标记清除。从表面上来看，它们都是关注某个量的可达性。如果发现一个量不再可达，就将其清除。

对于一个局部变量，编译器既有可能将它分配在堆上也有可能分配在栈上，而这一行为**并不是**由用不用new来决定的。

```go
var global *int

func f() {
    var dummy int
    dummy = 1
    global = &dummy
}
```

```go
func f() {
    y := new(int)
    *y = 1
}
```

以上示例的不同之处在于，第一块代码中dummy的地址暴露在了外侧（一个类似于全局变量上），这样函数执行完毕后，这个值的可达性仍然存在（称为dummy从f中**逃逸**了），那么编译器就会选择将这个值分配在堆上。第二块代码中的y显然在函数退出之后就不再可达，编译器可以将其分配在速度更快的栈上。

虽然说GC帮程序员完成了内存分配与回收的工作，但是在编写代码的时候我们仍然有必要去了解变量的内存是如何分配和运作的，这样有助于我们编写出高效的代码并且防止不必要的空间占用。例如对于一个不需要长期存在的量，就不要去维护一个外部的指针（更常见的情况是，将一个短期存在的量放在长期存在的结构体上，这种比较隐蔽），否则导致其可达性一直存在无法被释放，严重时则内存泄露（不断累积）。

## 元组赋值（Tuple Assignment）

元组赋值即在一个赋值语句的左右侧提供多个参数，这样一次性完成多个量的赋值。元组赋值总是先算右侧的表达式，然后将其赋值给左侧。借助元组赋值，可以完成值的交换（`x, y = y, x`），还可以简化一些算法的写法：
1. 最大公约数
```go
func gcd(x, y int) int {
    for y != 0 {
        x, y = y, x%y
    }
    return x
}
```
2. 斐波那契
```go
func fib(n int) int {
    x, y := 0, 1
    for i := 0; i < n; i++ {
        x, y = y, x+y
    }
    return x
}
```

一些函数例如`os.Open`，除了结果以外还会返回一个error或者布尔值（一般命名为`ok`）来表示是否出现了错误，此时就需要进行元组赋值或者短变量声明。除此之外，Go中有三个运算符会返回多个值（但这个是可选的，只有在左侧期待两个值的时候有这种行为）。

```go
v, ok := someMap[key]
v, ok := x.(T)
v, ok := <-ch
```

## 可赋值性（Assignability）

- 显式赋值：赋值语句
```go
a := 1
```

- 隐式赋值：例如使用组合字面量、map的值以及传入channel的值等
```go
medals := []string{"gold", "silver", "bronze"}
```

一般来说只要两个变量的类型是相同的，那么右值就可以赋值给左值。而nil始终可以赋值给一个指针或者接口类型。

两个值的可比性（是否可以用`==`和`!=`来比较）与其可赋值性有关系，例如任意两个相互比较的值必须也互相可赋值，反过来两个可赋值的值也应该可以相互比较。

## 类型定义

`type`用于定义一个命名类型（named type），被命名的那个类型称为底层类型（underlying type）。这种方式可以用来定义一些底层类型相同（例如都是`float64`）但是其含义并不相同的值的类型，例如摄氏度和华氏度。

- 一个类型T的类型转换写作`T(x)`，一般仅当两个类型的底层类型相同，或者两个指针指向的值的底层类型相同时，可以进行转换。这种转换不会改变这个值在底层的表示，例如都是`float64`。
- 数值类型、字符串、切片之间也可以进行转换，这些转换会改变值的底层表示，例如将一个`float64`转换为`int`后，其小数部分会被丢弃；将一个`string`转化为`[]byte`时会分配内存以存储该字符串的拷贝。

> In any case, a conversion never fails at run time. (p40)

一个命名类型的底层类型决定了这个值的结构（structure）、表示（representation）以及所支持的各种操作（supported operations）。例如底层类型为`float64`的命名类型可以和`float64`一样进行算术运算。

比较运算符（`==`、`>` 等）可以比较两个相同的命名类型的值，或者与一个具有相同底层类型的非命名类型。两个不同的命名类型的值不能进行比较。

```go
type Celsius float64
type Fahrenheit float64

var c Celsius
var f Fahrenheit
fmt.Println(c == 0) // 可以将命名类型与相同底层类型的非命名类型比较
fmt.Println(f == 0)
fmt.Println(c == f) // compile error
fmt.Println(c == Ceilsius(f)) // true
```

注意`T(x)`不会改变值的结构或表示，只改变类型。`c == Ceilsius(f)`为true是因为c和f的值同时为0，实际上这样的比较是不恰当的。

## 包的初始化（Package Initialization）

```go
var a = b + c // (3)
var b = f() // (2)
var c = 2 // (1)

func f() int {
    return c + 1
}
```

包的初始化过程中，会对顶层变量按照它们的声明顺序进行计算和赋值。如果一个变量依赖了另一个变量，则声明顺序会略有区别。为一个包顶层变量赋值的表达式称为**初始化表达式（initializer expression）**。初始化表达式在部分情况下不是很好用，例如当这个值需要比较复杂的控制流程、循环来初始化时。

一个包内可以有若干个名为`init`的初始化函数。这个函数不能被调用也不能被引用，它会在包初始化的时候被调用，可以用来实现对包顶层变量的赋值。

```go
package popcount

// pc[i]是i的population count（汉明权重）
var pc [256]byte

func init() {
    for i := range pc {
        pc[i] = pc[i/2] + byte(i&1)
    }
}
```

同一个包里的初始化表达式以及`init`的整体调用顺序与其所在文件被传递给编译器的顺序有关。`go`指令会将文件按照字母顺序排序传递给编译器。

包被初始化的顺序取决于其在程序中的导入顺序，作为依赖的包会早于导入该依赖的包初始化，因此main包总是最后初始化。

> One package is initialized at a time, in the order of imports in the program, dependencies first, so a package `p` importing `q` can be sure that `q` is fully initialized before `p`’s initialization begins. (p44)


## 作用域（Scope）

|概念|含义|特点|
|:-:|:-:|:-:|
|作用域（scope）|编写程序过程中可以访问到该标识符的范围|compile time property|
|寿命（lifetime）|运行期间该标识符可以被程序其它地方使用的期限|runtime property|

- 语法上的**块（block）**是使用花括号括起来的一段语句。在块内声明的标识符无法被块外面访问到。
- 将一系列标识符组成的某个足以区分作用域的范围称为一个词法块（lexical block）。一整个程序、一整个包、一整个文件以及每一个for、if、switch语句、switch语句和select语句的每一个case，以及每一个block都有其对应的词法块，最大的那一个词法块称为全局块（universe block）。

|对象|所处的词法块|访问范围|
|:-:|:-:|:-:|
|内置的类型和常量|universe|整个程序的各个位置|
|包的顶层（即函数的外部）定义的变量等|package level|同一个包的任何位置|
|通过import导入的包|file level|同一个文件的任何位置|
|控制流标签`label:`|*function level*|包围它的那个函数|
|函数中声明的标识符|local|函数内部的某个部分|

只要所处的词法块不同，程序中可以反复定义多个同名的标识符。例如，你可以定义一个local的标识符名为new来覆盖掉位于universe block的new，因为它们不在一个词法块。重定义的标识符的作用域不宜过大，否则可能会干扰代码理解。

当编译器遇到一个标识符时，它将从最内侧的词法块开始向外寻找第一个匹配，直到达到全局块。如果位于内侧的词法块与外侧的词法块包含了同名的标识符，内侧的会被采用，此时称内侧的标识符**遮蔽（shadow）**了外侧的标识符。

```go
func f() {

}

func main() {
    f := "f"
    fmt.Println(f) // "f"
}
```

词法块不一定和block一样是由花括号括起来的。for循环会创建两个词法块：
1. 其循环体（是一个block）确定的一个显式的词法块
2. 以及由初始化语句确定的一个隐式block

后者中定义的变量可以在for循环的条件、post statement和循环体中访问。

```go
func main() {
    x := "hello"
    for _, x := range x {
        x := x + 'A' - 'a' // RHS的x是for定义的，LHS遮蔽RHS的x
        fmt.Printf('%c', x) // "HELLO"
    }
}
```

if语句也会创建这种“隐式”的词法块。在if中，跟在一个分支后的分支可以看到其之前分支花括号前定义的那些标识符，例如下面的`else if`可以看到第一个`if`定义的x；`else`的body可以看到前两个分支中定义的`x`和`y`，而if外部则不可以。需要注意body内部的不会被外部看到。

```go
if x := f(); x == 0 {
    // ...
} else if y := g(x); x == y {
    // ...
} else {
    fmt.Println(x, y)
}
```

switch类似，其语句的条件部分和case body各会创建一个block。

**在package level**，作用域不受变量定义的顺序影响，这使得我们可以写递归函数以及递归类型。

### 短变量声明

在使用短变量声明时需要额外注意作用域。

```go
var cwd string
func init() {
    cwd, err := os.Getwd()
    if err != nil {
        log.Fatalf(/* ... */)
    }
}
```

以上程序无法通过编译，因为`cwd, err := os.Getwd()`实际上不仅在init中声明了err，还声明了一个local的cwd，它遮蔽了package level的cwd，产生unused variable。如果为了日志的全面性，在函数的最后加上`log.Printf("...", cwd)`，逻辑上可能会误认为这里是全局变量，而其实这里用的是local，到这个时候编译器也检查不出来这个错误了。
