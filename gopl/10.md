# Ch10. Packages and the Go Tool

本章主要介绍Go语言中的包管理、常用package以及go命令行工具。

## Go编译速度快的三点原因

1. 每个文件都必须在开头显式地给出导入的package，这样就不需要后续再查找
2. 各个package之间的导入关系构成一张有向无环图（DAG），使得并行编译成为可能
3. 每个package的编译中间产物还包含了其导入的内容，使得编译过程无需递归读取调用链上的所有内容

## 导入路径

导入路径（import path）是package的标识符。Go语言规范没有规定这些字符串的具体格式，但根据现有工具链，导入路径仍有convention需要遵循。

如果package需要对外共享，为了防止冲突，其命名空间应该是全局唯一的，也就是不与其它任何package冲突。要做到这一点，可以使用类似于Java的网域命名法。这里域名可以是你所有的域名，或是托管该代码服务的域名，例如`github.com/YourName/YourProject`。

公开发布的非标准库package应该使用网域命名，在保证唯一性的同时，也方便了对仓库的查找。

## package定义

每一个Go程序的开头都必须是一个package定义语句，该语句的作用是为当前文件所在的package定义一个默认的名称。称为默认是因为其它包在导入package的时候可以为其定义别名。

通常导入路径的最后一部分是包名，例如导入路径为`math/rand`或`crypto/rand`的package的包名都为rand，在程序内部都用`rand.Something`来访问。该惯例有三个例外：

1. 任何一个需要被编译成可执行文件的package的包名只能是main
2. 名称以`_test`结尾的Go原文件的包名可能也有`_test`后缀，这个时候同一个目录下除了有原有的package外，还定义了一个外部测试包（external test package）。引入两种package的目的是解决在编写测试过程中有的时候不可避免引入的导入循环问题
3. 有的时候为了管理版本，我们在import path最后加上一个版本号，例如`gopkg.in/yaml.v2`，但包名仍然沿用原本的yaml

## 导入

import语句总是存在于package语句和第一个非import语句之间。import语句有单行import或用括号括起来的多行import，它们是等价的，常使用后者。在后者的形式中，导入的package名称是没有顺序的，但gofmt、goimports工具一般会将其按照字母序排序。

为了解决上面提到的不同import path相同包名的问题，可以使用重命名导入（renaming import）。

```go
import (
    "crypto/rand"
    mrand "math/rand"
)
```

重命名导入不仅适合在冲突场景下使用，还适合在一些包名并不适合直接使用的代码中使用，例如自动生成的SDK代码。这里的一个例子是阿里云SDK中的一系列代码：
```go
import (
    ecs "github.com/alibabacloud-go/ecs-20140526/v7/client"
    vpc "github.com/alibabacloud-go/vpc-20160428/v6/client"
    openapi "github.com/alibabacloud-go/darabonba-openapi/v2/client"
)
```

在一些情况下，我们需要导入一个包，但并不需要真的用到其中的内容。例如我们可能会希望执行这个包中的init函数和package level的变量初始化语句。但导入一个包而不使用是一个compile error（类似于unused variable）。我们可以将其重命名为`_`来实现没有名称的导入。

```go
import _ "image/png" // register PNG decoder
```

这种导入被称为一个空导入（blank import）。

一个空导入作用的具体例子是`image/png`包，它的init函数中注册了对PNG图像的支持（调用`image.RegisterFormat`），因此在导入这个包之前，`image.Decode`函数不具有对png图片的支持。如果我们要编写一个将其它格式的图像转换为JPG格式的程序，我们会想到要导入的包是`image`和`image/jpeg`，但不包括`image/png`。这使得`image.Decode`无法解析PNG。这个时候使用空导入就可以解决问题。

类似的还有数据库的驱动程序，我们不需要真的使用里面的函数，但我们需要这个注册的过程。

```go
import (
    "database/sql"
    _ "github.com/lib/pq" // postgres
    _ "github.com/go-sql-driver/mysql" // mysql
)
```

## 命名惯例

### 包名

- 尽量使用较短的名称，但不要太短以至于有点像密码
- 尽量包含信息，减少模糊，例如比起util，使用具有特定主体的名称，如imageutil、ioutil更好
- 避免使用与本地变量重名的包名，例如path、last、next、width、height
- 一般使用单数形式。标准库中的bytes、errors、strings是为了避免冲突
- 避免挪用共识，例如用temp来代替temperature是不好的，因为temp是temporary的常用缩写

### 成员名

> When designing a package, consider how the two parts of a qualified identifier work together, not the member name alone.

当我们需要指代一个包中的成员的时候，我们需要同时带上包名和成员名，它们共同贡献了这个标识符的复杂度。在设计成员名的时候，我们应该考虑充分利用这份复杂度，让包名和成员名搭配起来。例如
- `bytes.Equal` 代指比较两个`[]byte`的函数
- `flag.Int` 代指int类型的命令行参数创建函数
- `http.Get` 代指发HTTP GET请求的函数
- `json.Marshal` 代指JSON序列化函数

以及strings包中有Index、Replacer、NewReplacer、Reader、NewReader等，这些成员的名称没有一个包含了“string”的字样。因其已经在strings包中，其作用仍然是明确的。

对于一个单类型包（single-type package），即只暴露出一个类型，功能都在这个类型的方法上完成的包，其中的类型成员名一般重复包名（例如`rand.Rand`、`template.Template`），用于返回该类型的一个新实例的函数一般就叫做New。
