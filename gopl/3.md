# Ch3. Basic Data Types

Go的基础数据类型包括数值类型、字符串和布尔类型。

## 整型

Go中提供带符号和不带符号两种整型，每一种整型都有8、16、32、64四种大小。

|类型|含义|备注|
|:-:|:-:|:-:|
|`int8`, ...|有符号整型||
|`uint8`, ...|无符号整型||
|`int`|未指定大小的有符号整型|1|
|`uint`|未指定大小的无符号整型|1|
|`rune`|字符型，用于表示一个Unicode码点|2|
|`byte`|字节类型，用于表示一个字节的数据|3|

注：
1. `int`和`uint`这两种类型的具体大小取决于平台，最终取决于编译器。可以确定的是它们有着相同的大小，但不能直接认为`int`和`uint`是32位或者64位的。
2. `rune`类型与`int32`等价，用于表示一个Unicode码点值。
3. `byte`类型与`uint8`等价，用于表示一段raw data而不是一个数值。
4. 不同的数值类型之间不能直接赋值，需要进行转换。

### 有符号整型

在Go中，有符号整型使用补码来表示，最高位用于表示其符号。一个n位的有符号数的取值范围是-2^(n-1) ~ 2^(n-1)-1。无符号整型的取值范围是0 ~ 2^n-1。

### 整型运算

- `%` 只能用于整型。对于负数之间的取余运算，在Go中规定余数的符号总是与被除数相同：`-5%3==-5%-3==-2`。
- 对于 `/`，整型的除法的结果约至0（相当于丢掉小数部分）：`-5/2=-2`
- 在运算过程如果发生了溢出，超过规定位数的高位会被丢弃。这个过程可能影响到数字的符号位，会导致有符号数的符号的改变。
- Go中的数值类型和字符串都是可比的（comparable）、有序的（ordered），这表示对于同类型的值，它们可以使用`==`、`!=`以及大小比较运算符来进行比较。
- 一元运算符`+`和`-`对于整型的作用
  - `+x`相当于`0+x`
  - `-x`相当于`0-x`
- 对于浮点型和复数型的作用
  - `+x`相当于`x`
  - `-x`是`x`的负（negation）

### 位运算符

|运算符|含义|
|:-:|:-:|
|`&`|按位与|
|`\|`|按位或|
|`^`|按位异或（二元）<br/>按位非（一元）|
|`&^`|按位清除|
|`<<`|算术左移（对于有符号）<br/>逻辑左移（对于无符号）|
|`>>`|算术右移（对于有符号）<br/>逻辑右移（对于无符号）|

按位清除运算符的效果如下：
```go
z := x &^ y
// z是结果，x是原始值，y是清除参照
```
```
    x 0010 1001 1101 1100
&^  y 1111 1111 0000 0000
-------------------------
    z 0000 0000 1101 1100
```

### 位运算符例子：bitset

设用二进制数$a_7a_6a_5a_4a_3a_2a_1a_0$来表示集合$\set{i|a_i=1}$。

```go
var x uint8 = 1<<1 | 1<<5
// x = 0010 0010 表示集合 {1,5}
var y uint8 = 1<<1 | 1<<2
// y = 0000 0110 表示集合 {1,2}
```

此时可以直接用位运算来表示集合运算。

|位运算|对应的集合运算|
|:-:|:-:|
|`&`|交集|
|`\|`|并集|
|`^`|对称差|
|`&^`|差|

|表达式|结果（`fmt.Printf("%08b")`）|解释|
|:-:|:-:|:-:|
|`x&y`|00000010|$\set{1,5}\cap\set{1,2}=\set{1}$|
|`x\|y`|00100110|$\set{1,5}\cup\set{1,2}=\set{1,2,5}$|
|`x^y`|00100100|$\set{1,5}\Delta\set{1,2}=\set{2,5}$|
|`x&^y`|00100000|$\set{1,5}-\set{1,2}=\set{5}$

> `08` modifies `%b`(an **adverb!**) to pad the result with zeros to exactly 8 digits. (p54)

作者在这里提到`08`可以算是一个“副词”（adverb），因为在Go中用于格式化输出的占位符被称为“动词”（verb），`08`是用来修饰动词`%b`的，所以它就是个副词了。

### `uint` is not intended for non-negative values

虽然Go提供了无符号数值类型，但是对于数组的长度（不可能为负）通常还是使用`int`。例如`len`返回的就是一个`int`，它返回`uint`反而不合适，将破坏下面常见的倒序for循环写法，`i>=0`永远成立，且本应破坏循环条件的最后一个`i--`将导致`i`负溢出成为最大值，导致下标越界panic。

```go
for i = len(arr) - 1; i >= 0; i-- {
    // ...
}
```

因此无符号整型一般只用于需要其特殊运算性质（逻辑移位，例如bitset）、用于解析二进制文件、哈希以及加密时；无符号整型并不简单代表适合用于“不可能为负”的那些数值。

### 数值类型转换

根据只有相同的类型才能做运算或者赋值的原则，一些数值往往需要进行一些转换。这些转换并不会改变数值的底层表示，其意义只在于提示编译器。

```go
var orange int32 = 1
var apple int16 = 2
var total int = int(orange) + int(apple)
```

但位数不同的整型之间或者浮点数与整型之间的类型转换也有可能导致值的改变。

```go
var orange int32 = 222
var total int8 = int8(orange) // -34
```

下面的代码则无法通过编译。

```go
var smallContainer int8 = 256 // compile error: cannot use 256 (untyped int constant) as int8 value in variable declaration (overflows)
```

```go
fmt.Println(int8(444)) // compile error: constant 444 overflows int8
```

应当尽可能避免这种会导致数值变化的转变，因为其行为并不是确定的，而是取决于编译器。

### 数值输出

数值字面量可以写成十进制、`0x`开头的十六进制以及`0`开头的八进制形式。在输出的时候可以使用动词来控制输出的基数
|动词|含义|`0666`字面量的输出|
|:-:|:-:|:-:|
|`%d`|十进制数值|438|
|`%o`|八进制数值|666|
|`%#o`|带前缀的八进制数值|0666|

类似的，`%#x`是带`0x`前缀的十六进制数值，`%#X`是带`0X`前缀的大写十六进制数值。另外，还可以在`%`后面`#`（如有）前面加上`[n]`来指代Printf传入的第n个值参数。

```go
// 只需要传入一个参数，就可以输出四种格式
fmt.Printf("%d %[1]x %#[1]x %#[1]X\n", int64(0xdeadbeef))
// 3735928559 deadbeef 0xdeadbeef 0XDEADBEEF
```

rune类型的字面量通常直接写成用单引号引起来的单个字符（可以是Unicode）。用`%c`可以打印出字符的内容，`%q`则是带引号的字符。

```go
ascii := 'a'
fmt.Printf("%d %[1]c %[1]q\n", ascii)
```

注意这里的ascii被认为是一个rune类型的值，在这里写成
- `var ascii int32 = 97`
- `var ascii rune = 97`

均可，但是`ascii := 97`不可以（会被默认为int类型，无法用`%c`、`%q`打印）。

## 浮点型

Go的浮点型数值的运算性质遵从IEEE 754标准，提供`float32`和`float64`两种大小的浮点型类型。math package下的`math.MaxFloat32`, ...常量分别提供了两种类型的最大值和最小值。

### 通常应该使用float64

float32数值的精度可以认为是十进制下小数点后约6位；float64则约为15位。通常情况下应该使用float64，因为float32数值的计算很容易积累误差，且从16777216开始float32便无法精确表示整数。
```go
val := float32(1<<24) // 16777216（1677万）是第一个无法被float32精确表示的正整数
fmt.Println(val == val+1) // true
```

浮点型数值的字面量可以根据需要省略整数部分或者小数部分，但是不能省略小数点。

```go
v1 := .707
v2 := 1.
fmt.Printf("%t %t\n", v1, v2) // float64 float64
```

### 浮点型数值输出

```go
for x := 0; x < 8; x++ {
    fmt.Printf("x = %d e^x = %8.3f\n", x, math.Exp(float64(x)))
}

// x = 0 e^x =    1.000
// x = 1 e^x =    2.718
// x = 2 e^x =    7.389
// x = 3 e^x =   20.086
// x = 4 e^x =   54.598
// x = 5 e^x =  148.413
// x = 6 e^x =  403.429
// x = 7 e^x = 1096.633
```

|动词|含义|123.456789|123456789.|0.00000123456789|1.|
|:-:|:-:|:-:|:-:|:-:|:-:|
|`%f`|1|123.456789|123456789.000000|0.000001|1.000000|
|`%.8f`|2|123.45678900|123456789.00000000|0.00000123|1.00000000|
|`%8.3f`|3| 123.457|123456789.000|`___0.000`|`___1.000`|
|`%e`|4|1.234568e+02|1.234568e+08|1.234568e-06|1.000000e+00|
|`%.2e`|5|1.23e+02|1.23e+08|1.23e-06|1.00e+00|
|`%g`|6|123.456789|1.23456789e+08|1.23456789e-06|1|

1. 普通（十进制）方式表示，默认保留6位
2. 普通，保留8位
3. 普通，保留3位，且数值位数为8，不足的用空格填充
4. 科学计数法，默认保留6位
5. 科学计数法，保留2位
6. 根据数值大小决定一个具有足够精度且最紧凑的表示方式（常用）

### IEEE754特殊数值

- `±x/0`产生±Inf
- `0/0`产生NaN

NaN与任何数值的任何比较（除了`!=`）都会返回false，`!=`总是返回true。可以使用`math.NaN()`创建该值，但它不应该用于在一般的计算中作为某种“哨兵”（sentinel value）。更好的方式是直接用布尔值或者error去清晰地表达计算过程中出现的异常情况。

## 复数

Go提供`complex64`和`complex128`两种类型用来表示复数，其底层分别对应float32和float64。

- `complex(a, b)`内置函数可用于创建一个复数a+bi，也可以直接使用字面量来创建
- `real(z)`可以获取其实部
- `imag(z)`可以获取其虚部

如果一个浮点数值字面量或者整数字面量后面紧跟了一个i，那么它将变为一个虚数字面量（imaginary literal），也就是表示一个纯虚数。

`math/cmplx` package下面提供了复数相关的数学函数，如`cmplx.Sqrt`可用于计算$\sqrt{-1}$。

## 字符串

> A string is an immutable sequence of bytes. (p64)

字符串中可以包含任意的数据，例如值为0的字节，但通常其内容是人类可读的文本
- 字符串默认使用UTF-8进行编码
- len函数返回一个字符串所占的**字节**数量
- 以下标访问字符串，得到的是对应位置的那个字节

子串可以使用`s[i:j]`的语法获取，所得子串长度为j-i。i不填默认为0，j不填默认为len(s)。若i、j越界或者j < i会触发panic。

字符串之间可以使用`==`、`>`等运算符进行逐字节比较。当比较字符串大小时，依据是该字符串每一位的字典序（lexicographic ordering）。

### 字符串是不可变的

字符串的值不可变，不能单独地为字符串的某个字节进行赋值。当使用`+=`等赋值运算符尝试改变一个字符串变量的值时，会创建一个新的字符串。

```go
s := "hello"
t := s
s += "world" // 现在s不再存储原本的字符串
```

字符串的不可变性使得字符串的内存可以安全地共享。在这一前提下，提取一个字符串子串的语法并没有复制该字符串，也没有分配新的空间，这使得取子串的操作是高效且廉价（cheap）的。并且类似于上面代码中`t := s`的操作可以理解为对该字符串的复制（与真正的克隆不一样），也是廉价的（因为相当于只复制了引用）。

### 字符串字面量

Go的源文件总是使用UTF-8编码，Go中的字符串也默认使用UTF-8编码，使得我们可以在字符串字面量中直接使用Unicode字符来表示码点，除此之外：
- 可以使用`\`开头的转义序列来插入特殊的字节，例如`\n`、`\t`等
- 可以使用`\xhh`，其中`hh`是两位十六进制数，不区分大小写，来表示十六进制转义字符
- 可以使用`\ooo`，其中`ooo`是三位八进制数，来表示八进制转义字符，注意不可超过`\377`

### 原始字符串字面量

可以使用反引号来表示一个原始的字符串字面量（raw string literal），在这种字面量中不会考虑转义序列。唯一处理的是会将carriage return（`\r`）自动删除掉，使得该字符串在所有的平台上都显示一致。

原始字符串字面量通常用于编写大段提示文字、JSON、HTML模板等。

### Unicode

在Unicode中为每一个独立的字符分配的编号被称为一个Unicode码点，在Go中这个码点被叫做rune（符文、符号）。

第八版Unicode为一百多种语言的一万两千多个字符分配了码点，我们可以使用一个int32来存储码点，那么字符串就是int32的序列。这种编码方式称为UTF-32。但是一些字符甚至很多字符并不需要int32的空间，且最常用的字符不超过65536个，int16的空间就已经足够。为了解决空间的浪费，Ken Thompson和Rob Pike（**他们也是Go的创造者**）提出了UTF-8这种变长编码方式，每一个字符可能占据1~4个字节，且ASCII字符只占据一个字节，大多数常用字符只占据2~3个字节。

|二进制表示|码点范围|注释|
|:-|:-:|:-:|
|0xxxxxxx|0-127|ASCII|
|110xxxxx 10xxxxxx|128−2047|不使用小于128的值|
|1110xxxx 10xxxxxx 10xxxxxx|2048-65535|不使用小于2048的值|
|11110xxx 10xxxxxx 10xxxxxx 10xxxxxx|65536−0x10ffff|不使用范围外的值|

注：110、10、1110等等不参与实际数值的计算；实际数值只看`x`部分。

UTF-8的变长特性导致不能直接获取一个字符串的第n个字符，但带来了许多优势：
- 节省空间
- 与ASCII保持兼容性
- 自同步（self-synchronizing）：至多跳过三个字节就可以找到一个字符的起点，且跳过的依据是确定的：以`10`开头的字节不可能是字符的起点
- 前缀码（prefix code）：这使得Unicode可以自左向右解析，无需前瞻也没有二义性
- 无重叠编码：对任意一个Unicode码点，不存在一个Unicode码点的字节包含于或包含它的字节，这使得搜索字节时无需考虑上下文
- 除了U+0000外，其余Unicode码点不包含0x0字节，这使得任意一个码点不会意外包含NUL字符，方便了编程语言对字符串的解析

正因为Unicode是无重叠的编码，在Go中可以这样实现HasPrefix和HasSuffix

```go
func HasPrefix(s, prefix string) bool {
    return len(s) >= prefix && s[:len(prefix)] == prefix
}

func HasSuffix(s, suffix string) bool {
    return len(s) >= suffix && s[len(s)-len(suffix):] == suffix
}
```

### Unicode用于字面量中

字符串字面量中可以使用`\uhhhh`来表示一个16位的Unicode码点或者用`\Uhhhhhhhh`来表示一个32位的Unicode码点（后者少见）。以下四行是等价的。

```go
"世界"
"\xe4\xb8\x96\xe7\x95\x8c"
"\u4e16\u754c"
"\U00004e16\U0000754c"
```

一个字符字面量中也可以使用`\u`和`\U`的写法，但`\xhh`写法只允许用于小于256的码点值。

### 处理Unicode码点字符串

可以使用`utf8.RuneCountInString(s)`来获取字符串中包含的码点个数：
```go
s := "Hello, 世界"
fmt.Println(len(s)) // "13"
fmt.Println(utf8.RuneCountInString(s)) // "9"
```

为了遍历这样一个字符串，可能需要如下循环。在这里我们需要自己在循环中根据size来更新i的值，而不能让其自然加一。

```go
for i := 0; i < len(s); {
    r, size := utf8.DecodeRuneInString(s[i:]) // 解码出字符串中包含的第一个rune
    i += size
}
```

为了简化，我们可以用range。range关键字在遍历字符串时会隐式地对字符串进行解码，其下标会自动更新到正确的字符开头。例如下面的循环遍历到“世”时输出的i为7（表示“世”的码点的第一个字节在该字符串下标为7的位置），到“界”时输出的i为10。

```go
for i, r := range "hello, 世界" {
    fmt.Printf("%d\t%q\t%d\n", i, r, r)
}
```

我们甚至可以直接实现`utf8.RuneCountInString`（它确实是这样实现的）：

```go
cnt := 0
for range "hello, 世界" {
    cnt++
}
```

### 非Unicode编码行为

range关键字的自动解码行为基于Go中字符串使用的编码为Unicode这一前提。如果字符串中没有包含正确的UTF-8编码数据，`utf8.DecodeRuneInString(s)`和range不会报错，但会生成一个用于表示错误的字符（称为替换字符，replacement character）`\uFFFD`（�）。这个替换字符是package utf8中的一个常量`utf8.RuneError`，见于`DecodeRuneInString`的文档：

> ... If s is empty it returns (**RuneError**, 0). Otherwise, if the encoding is invalid, it returns (**RuneError**, 1). Both are impossible results for correct, non-empty UTF-8.

> When a program encounters this rune value, it’s often a sign that some upstream part of the system that generated the string data has been careless in its treatment of text encodings. (p70)

### `[]rune`、`string`与数值的类型转换

一个string可以被类型转换为rune数组，其结果是字符串中包含的各个Unicode码点的编码值。

```go
s := "プログラム"
r := []rune(s)
fmt.Printf("%x\n", r) // "[30d7 30ed 30b0 30e9 30e0]"
```

反过来，将一个`[]rune`转换为string，其结果是数组中所有rune拼合成的字符串。

将一个数值转换为string，相当于将该数值看成一个Unicode码点。

```go
fmt.Println(string(0x4eac)) // "京"
fmt.Println(string(1234567)) // "�"
```

### 字符串与字节数组

在字符串操作方面，以下四个package很重要
- `bytes`用于操作`[]byte`。由于字符串是不可变的，逐步构造一个完整的字符串会涉及到许多内存的分配和复制，此时使用byte相关操作会更高效
- `strconv`提供将布尔值、整型、浮点型转换为其对应的字符串表示以及反过来的辅助函数，还包括用于给字符串加引号与取消引号的函数
- `unicode`提供针对rune的各种操作和判断
- `strings`提供针对字符串的各种操作和判断

字符串与`[]byte`可以互相转换，无论是哪种转换，过程都会发生内存的重新分配和复制。
```go
s := "abc"
b := []byte(s)
s2 := string(b)
```

`[]byte`和字符串很像，它们之间会有一些相同作用的方法，为了避免使用这些方法之前需要进行上面提到的昂贵的转换过程，`strings`和`bytes` package里有着相同作用的一系列函数，例如`func Join(a []string, sep string) string`和`func Join(s [][]byte, sep []byte) []byte`，以及`HasPrefix`、`HasSuffix`等等，它们唯一的区别是类型不同。

`bytes.Buffer`提供了一个高效地构造和编辑`[]byte`的方式（同时也可以用于构造字符串，如果直接使用string，由于不可变性效率会很低）可以将字符串、字符、字节或者`[]byte`写入这个Buffer。在写入字符的时候最好使用`WriteRune`方法而不是`WriteByte`，后者只适合ASCII字符。

```go
// intsToString is like fmt.Sprint(values) but adds commas.
func intsToString(values []int) string {
    var buf bytes.Buffer
    buf.WriteByte('[')
    for i, v := range values {
        if i > 0 {
            buf.WriteString(", ")
        }
        fmt.Fprintf(&buf, "%d", v)
    }
    buf.WriteByte(']')
    return buf.String()
}

func main() {
    fmt.Println(intsToString([]int{1, 2, 3})) // "[1, 2, 3]"
}
```

### 字符串与数值之间的转换

将一个整型转换为字符串可以使用`fmt.Sprintf`或者`strconv.Itoa`（integer to ASCII）。

`FormatInt`和`FormatUint`可用于将数值以指定基数转换为字符串，但通常`%b`、`%d`、`%x`、`%o`等动词更方便。

将一个字符串转换为整型可以使用`strconv.Atoi`、`strconv.ParseInt`、`strconv.ParseUint`。`Atoi`的返回值是一个int，Parse...返回的是(u)int64。Parse...的第三个参数表示转换得到数值的位数，例如16表示该值的大小与(u)int16类型相符合。特殊值0表示平台类型(u)int。指定该参数不会改变其返回值类型为(u)int64。

## 常量

> Constants are expressions whose value is known to the compiler and whose evaluation is guarateed to occur at compile time, not at run time. (p75)

常量是指在编译期就一定能够确定的量。常量必须是基本数据类型：字符串、布尔值或者数值类型。

常量的计算可以在编译期就完成。编译器往往可以识别一个包含常量操作数的表达式中所包含的错误，例如除以0、下标越界、产生非有穷量的浮点数运算等，还包括前面涉及到的`int8(999)`问题。

> The results of all arithmetic, logical, and comparison operations applied to constant operands are themselves constants, as are the results of conversions and calls to certain built-in functions such as `len`, `cap`, `real`, `imag`, `complex`, and `unsafe.Sizeof`. (p76)

常量与常量的算数、逻辑、比较操作的结果依然是常量，以及以上列出的这些函数传入常量得到的返回值也是常量。

由于常量是编译期就已经掌握的信息，它可以出现在类型中，例如指定一个slice的长度。

```go
const IPv4Len = 4
var p [IPv4Len]byte
```

### 成组常量定义

可以一次性定义多个常量。在同一个组中，如果一个常量除了标识符以外的部分省略，则默认沿用其之前的那个常量的类型和值。

```go
const (
    a = 1
    b // 相当于 = 1
    c = 2
    d
)
```

但其实这样的作用不大。在实际中，更经常用到的是一种常量生成器（constant generator）`iota`。


```go
type Weekday int
const (
    Sunday Weekday = iota // = 0
    Monday // = 1
    Tuesday // ...
    Wednesday
    Thursday
    Friday
    Saturday
)
```

以这种形式定义的成组常量便是Go中的**枚举（enumeration）**。`iota`可以放在一个表达式中：

```go
type Flags uint

const (
    FlagUp Flags = 1 << iota // 1<<0
    FlagBroadcast // 1<<1
    FlagLoopback // 1<<2
    FlagPointToPoint // ...
    FlagMulticast
)
```

这样构造出的Flag可以使用位运算来验证。
- 判断是否设置了一个flag
```go
func IsUp(v Flags) bool {
    return v & FlagUp == FlagUp
}
```
- 判断是否设置了多个flag
```go
func IsCast(v Flags) bool {
    return v & (FlagBroadcast|FlagMulticast) != 0
}
```
- 取消某个flag
```go
func TurnDown(v *Flags) {
    *v &^= FlagUp // 位清除
}
```
- 设置某个flag
```go
func SetBroadcast(v *Flags) {
    *v |= FlagBroadcast
}
```

另外一个使用`iota`的例子

```go
const (
    _ = 1 << (10 * iota)
    KiB // 1024
    MiB // 1048576
    GiB // 1073741824
    TiB // 1099511627776（从这里超过1<<32）
    PiB // 1125899906842624
    EiB // 1152921504606846976
    ZiB // 1180591620717411303424（从这里超过1<<64）
    YiB // 1208925819614629174706176
)
```

由于Go中没有用来表示指数/乘方的运算符，所以无法使用`iota`来自动生成1000的n次方（KB、MB、...）

### 无类型常量

在Go中有很多常量没有被赋予具体的类型，对于这些无类型的常量，编译器会以极高的精度去表示，可以认为至少有**256位**的精度。共有六种无类型常量：
- 无类型的布尔值
- 整型
- 字符型
- 浮点型
- 复数型
- 字符串

Note: 此处严格来说应该是“未定义类型的（untyped）布尔值常量（boolean）”，等等。为了方便，直接说成“无类型布尔值”。

在上面的代码中，从`KiB`到`YiB`都没有设置类型，因此它们是无类型常量。从`TiB`开始，数值无法放进int32，从`ZiB`开始，数值无法放进int64，却可以正常地对它们做运算。

```go
fmt.Println(YiB/ZiB) // 1024
```

一旦某个常量被赋予了类型，那么它就不再具有极高的精度，并且在使用的过程中遵循类型的限制，如果要赋值给不同的类型需要转换。

```go
var Pi64 float64 = math.Pi
var Pi32 float32 = float32(Pi64)
```

`0`、`0.0`、`0i`、`'\u0000'`四个字面量都表示的是同一个值，但是因为语法不同，它们分别属于无类型整型、浮点型、复数型和字符型。在做运算的时候，正确选取字面量类型很重要，例如`/`运算对于整数和浮点型有不同的表现。

```go
var f float64 = 212
fmt.Println((f - 32) * 5 / 9) // 100，因为(f-32)*5是一个float64
fmt.Println(5 / 9 * (f - 32)) // 0，因为5/9是一个untyped int且值为0
fmt.Println(5. / 9. * (f - 32)) // 100，因为5./9.是一个untyped float
```

只有常量可以没有类型。当一个无类型常量被赋值给一个带有类型的变量时，常量的类型会隐式转换为对应的类型。
```go
var f float64 = 3 + 0i // untyped complex -> float64
f = 2 // untyped integer -> float64
f = 1e123 // ...
f = 'a'
```
效果上相当于为RHS都加上一个`float64(...)`。

将一个常量从一种类型转换为另一种类型时，需要确保另一个类型可以接受这个值（can represent the **original value**）。浮点数可以发生舍入。

```go
const (
    deadbeef = 0xdeadbeef
    f32 = float32(deadbeef) // 入
    f64 = float64(deadbeef) // 精确值
    i32 = int32(deadbeef) // compile error
)
```

如果将一个无类型常量赋给一个没有明确说明类型的变量时，这个变量的类型会被定为这个值对应的那个**默认类型**。

|常量类型|默认数据类型|
|:-:|:-:|
|整型|`int`|
|字符型|`rune`（`int32`）|
|浮点型|`float64`|
|复数型|`complex128`|


```go
i := 0 // untyped integer; implicit int(0)
r := '\000' // untyped rune; implicit rune('\000')
f := 0.0 // untyped floating-point; implicit float64(0.0)
c := 0i // untyped complex; implicit complex128(0i)
```
