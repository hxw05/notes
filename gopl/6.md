# Ch6. Methods

虽然Go中没有常见的继承、类等概念，但Go也是一门面向对象（OOP）的语言。关于何为面向对象编程，一直以来没有一个统一的定义。在这里的讨论中，规定
- 对象（object）是指一个有着方法的变量或者值
- 方法（method）是指一个附加在某个类型上的函数
- 如果一个程序使用方法来表示数据结构上的操作和属性（getter/setter/...）而不是让外部直接操作其内部表示，就称这个程序是面向对象的

## 方法声明

方法可以看作是函数的变形，通过在函数之前添加一个额外的参数——这个函数所附加的那个类型就在这个类型上定义了一个方法。这个添加的额外的参数被称为这个方法的**接收器（receiver）**，这种叫法来自于早期函数式语言将方法形容为“向对象发送信息”。在其他语言中，扮演同类作用的是`this`关键字（但是，JS中的`this`含义很多，不太一般；Python中用的是`self`，且这个`self`的获取方式与其他语言不太一样，反而跟Go有些相似）。Go中的接收器可以自行命名，如`(p Point)`或者`(pt Point)`，通常直接选择类型的首字母小写。

```go
type Point struct{ X, Y float64 }

// traditional function
func Distance(p, q Point) float64 {
    return math.Hypot(q.X-p.X, q.Y-p.Y)
}

// same thing, but as a method of the Point type
func (p Point) Distance(q Point) float64 {
    return math.Hypot(q.X-p.X, q.Y-p.Y)
}
```

### 选择器

用于指代一个类型上的方法的表达式被称为一个**选择器（selector）**，例如`p.Distance`，因为这个表达式为`p`这个`Point`类型的值选择了合适的方法。访问一个结构体上的字段的表达式也是选择器。为了避免混淆，不允许定义和结构体字段同名的方法。

Go与其他面向对象语言的另一个不同是，Go中在任意一个类型上面附加方法都是可行的，只要满足
- 它是一个命名类型，例如`type Path []int`
- 方法定义必须在该类型的同一个package下面
- 其底层类型不能是一个指针或接口

## 具有指针接收器的方法

由于函数调用是值传递的，当一个方法被调用时，其接收器也会像普通参数那样作为函数的参数被传入，也会被复制。如果希望避免复制，我们可以把方法定义在这个类型的指针类型上，这样方法的标识符就变为`(*Type).Method`。

需要注意的是，如果我们用一个表达式去表示这样一个方法调用，括号是不可省略的：`(*p).Method()`；`*p.Method()`会被解读为`*(p.Method)()`。

在实际开发中，**如果一个类型有具有指针接收器的方法，那么它的所有方法的接收器都应该是指针类型，即使根本不需要指针带来的特性**。

不能在一个指针类型上附加方法，而只能将方法的接收器定为指针类型。这是为了避免混淆。

```go
type P *int
func (P) f() { /* ... */ } // compile error: invalid receiver typ
```

### 语法糖

要调用一个具有指针接收器的方法，先获取类型的指针。下面的写法在一个指针上调用了`(*Point).ScaleBy`。
```go
r := &Point{1, 2}
r.ScaleBy(2)
fmt.Println(*r) // "{2, 4}"
```

相比于上面的写法，下面的方法就显得有些不优雅（ungainly），每次调用之前都需要刻意地获取指针。

```go
p := Point{1, 2}
pptr := &p
pptr.ScaleBy(2)
fmt.Println(p) // "{2, 4}"
```
```go
p := Point{1, 2}
(&p).ScaleBy(2)
fmt.Println(p) // "{2, 4}"
```

Go在这里提供了语法糖。如果调用时作为receiver的是一个**变量**，或者说一个可以获取地址的值，那么可以省略掉获取指针的这一步。
```go
p.ScaleBy(2)
```
而像`Point{1, 2}`（字面量）这样的值不能获取地址，也就不能使用该语法糖。

反过来，如果一个方法的receiver不是指针，而我们要在一个指针上调用这个方法，我们也可以省略对指针的解引用，`p.Method`将被自动转化为`(*p).Method`。

总结下来，一个合法的方法调用只有下列三种情况。为了表述方便，仿照函数中的常见叫法，我们将出现在形式定义里的receiver称为receiver形参（receiver parameter），而将实际调用表达式中的值称为receiver实参（receiver argument）。
- 实参与形参的类型完全相同
- 形参是指针类型`*T`，实参是一个类型为`T`的可以获取地址的值（i.e. 变量）
- 形参是值类型`T`，实参是一个指针`*T`

仅当一个类型的所有方法的接收器都是值类型而不是指针类型时，才能对该类型的值进行拷贝，否则无法确保其各个方法能正常工作。例如`bytes.Buffer`这个结构体类型就不能拷贝，否则会导致多个拷贝共享同一个底层字节数组，导致在使用过程中不可预测的行为。

### nil也可作为receiver实参

就像函数上指针类型参数位置可以接受nil的传入一样，一个方法的指针类型receiver也可以是nil，用于为这种类型（如map、slice）的零值提供操作。

例如IntList表示一个值全为int的链表，其链表项指向下一个链表项的指针Tail是一个`*IntList`，其零值为nil。在这里我们认为一个值为nil的`*IntList`表示一个空链表，其Sum方法返回0。

```go
// An IntList is a linked list of integers.
// A nil *IntList represents the empty list.
type IntList struct {
    Value int
    Tail *IntList
}

// Sum returns the sum of the list elements.
func (list *IntList) Sum() int {
    if list == nil {
        return 0
    }
    return list.Value + list.Tail.Sum()
}
```

为了可读性，当我们确实处理了nil作为receiver实参的情况时，最好在注释里面写清楚这个零值的具体含义，例如空链表。

再比如package url下的Values类型，底层是一个`map[string][]string`，可以称为一个多值字典（multimap）。在这里`Get`方法使用map的下标访问语法，根据长度去判断这个key对应的slice是否为空，如果不是空的，这个方法就返回空的字符串。这里虽然没有对nil的判断，但是利用了零值特性：
- 访问了不存在的key，获取到的值是`[]string`的零值nil
- 对nil调用len，返回0

于是在一个值为nil的Values上调用Get的结果就是一个空字符串。而如果调用Add，则会panic，因为它尝试更新一个nil map。

注意这里的方法中Values都是值形式。由于Values的底层类型是一个map，即使接收器没有写成指针形式，这些方法对map中值的修改对外部仍然是可见的（而不仅是局部修改）。其原理与slice类似，map和slice都可以看作是一个包含了指向其底层结构指针的结构体（map指向的是哈希表，slice指向的是数组），对一个map进行拷贝，并不影响其底层指针的指向，于是对map上的值做修改就会直接反映到原本的map里。当然，由于是拷贝的map，我们对这个map本身做赋值对外部是不可见的。

```go
// Values maps a string key to a list of values.
type Values map[string][]string

// Get returns the first value associated with the given key,
// or "" if there are none.
func (v Values) Get(key string) string {
    if vs := v[key]; len(vs) > 0 {
        return vs[0]
    }
    return ""
}

// Add adds the value to key.
// It appends to any existing values associated with key.
func (v Values) Add(key, value string) {
    v[key] = append(v[key], value)
}
```

## 使用结构体嵌入来实现类型组合

定义一个ColoredPoint用于表示一个带有颜色的点。我们可以写出三个字段，X、Y和Color。为了复用，我们考虑将之前的Color嵌入到ColoredPoint中来提供X和Y两个字段。嵌入除了提供字段外，还可以提供方法。

在这里之所以称为“提供”，是因为这种语法允许我们可以不访问这个嵌入的字段，而直接访问这个嵌入类型上的字段或方法，并且这个过程不受嵌入类型的可见性影响。

```go
import "image/color"

type Point struct{ X, Y float64 }

type ColoredPoint struct {
    Point
    Color color.RGBA
}

var cp ColoredPoint
cp.X = 1
fmt.Println(cp.Point.X) // "1"
cp.Point.Y = 2
fmt.Println(cp.Y) // "2"
```

```go
var p = ColoredPoint{Point{1, 1}, red}
var q = ColoredPoint{Point{5, 4}, blue}
fmt.Println(p.Distance(q.Point)) // "5"
p.ScaleBy(2)
q.ScaleBy(2)
fmt.Println(p.Distance(q.Point)) // "10"
```

虽然ColoredPoint上并没有定义Distance方法，但由于嵌入了Point，它也具有了Distance方法。这种现象称为Point的方法被提升（promoted）到了ColoredPoint中。通过这种方法，将多个类型嵌入到一个结构体中，可以获得一个具有多种方法的复杂类型。

### 组合与继承的区别

> Readers familiar with class-based object-oriented languages may be tempted to view Point as a base class and ColoredPoint as a subclass or derived class, or to interpret the relationship between these types as if a ColoredPoint “is a” Point. But that would be a mistake.

在这里我们用到了**组合（composition）**。参考Java、C++等语言，我们很容易将这里Point和ColoredPoint的关系理解为ColoredPoint*类似于*Point的一个子类，其继承了Point的属性和方法。但这样理解不太正确，一个体现就是在Distance的调用方式上：

```go
var p = ColoredPoint{Point{1, 1}, red}
var q = ColoredPoint{Point{5, 4}, blue}
fmt.Println(p.Distance(q.Point)) // "5"
```

我们不能将`p.Distance(q.Point)`简化为`p.Distance(q)`因为`Distance`这个方法的参数的确接收的是Point类型的值，而ColoredPoint**不是**一个Point。正确的理解是遵循组合/嵌入原本的语义，我们认为ColoredPoint上**持有**一个Point。对于那些提升的方法，可以简单理解为编译器为ColoredPoint生成了一系列wrapper。

```go
func (p ColoredPoint) Distance(q Point) float64 {
    return p.Point.Distance(q)
}
func (p *ColoredPoint) ScaleBy(factor float64) {
    p.Point.ScaleBy(factor)
}
```

这也表示，我们在一个ColoredPoint上面调用Point上的方法时，这些方法里面看到的receiver实参的类型也永远是一个Point，不可能是ColoredPoint，它们也就没有可能访问到ColoredPoint上的任何信息。

### 通过指针类型实现间接（动态）嵌入

匿名字段上的类型除了是值类型外，还可以是一个指针类型。此时指针类型的方法将会间接地被提升到其组合到的那个结构里。将匿名字段指定为指针类型的一个作用是，可以通过动态地修改组合类型上这个匿名字段的指向而修改组合类型的这部分底层数据。

```go
type ColoredPoint struct {
    *Point
    Color color.RGBA
}
p := ColoredPoint{&Point{1, 1}, red}
q := ColoredPoint{&Point{5, 4}, blue}
fmt.Println(p.Distance(*q.Point)) // "5"
q.Point = p.Point
p.ScaleBy(2)
fmt.Println(*p.Point, *q.Point) // "{2 2} {2 2}"
```

### 多个匿名字段

一个结构体上可以嵌入多个不同的类型。这些类型之间可能有着相同名称的方法。编译器在解析一个选择器（例如`p.ScaleBy`）的时候，以方法为例，按照自顶向下的顺序进行搜索，采用第一个搜索到的那个方法，即先搜索这个类型上直接定义的ScaleBy，然后搜索一次提升的ScaleBy（结构体上嵌入的类型），然后搜索提升两次的ScaleBy（结构体上嵌入的类型中嵌入的类型），以此类推。如果在同一级找到了两个相同的标识符，则报错。字段同理。

### 在未命名的类型上调用方法

方法定义的一个条件是其接收器的类型必须是一个具名类型，但是我们仍然可以通过嵌入来构造一个有着方法（但显然不是在它上面定义的）的unnamed type。下面的写法展示了将一个map和保护它的互斥锁关联在同一个struct中，然后以此类型定义一个cache的代码。在Lookup函数中，我们不再使用`mu.Lock()`的写法，而是直接在cache上调用Lock，因为`sync.Mutex`嵌入到了它的类型上。这种写法让代码更加符合语义：我们直接在cache本身上面上锁，隐去了底层使用互斥锁的这一细节（expressive name & self-explanatory syntax）。

```go
var cache = struct {
    sync.Mutex
    mapping map[string]string
} {
    mapping: make(map[string]string),
}
func Lookup(key string) string {
    cache.Lock()
    v := cache.mapping[key]
    cache.Unlock()
    return v
}
```

> The new variable gives more **expressive** names to the variables related to the cache, and because the sync.Mutex field is embedded within it, its Lock and Unlock methods are promoted to the unnamed struct type, allowing us to lock the cache with a **self-explanatory** syntax.

## 方法值、方法表达式

### 方法值

和函数一样，方法也可以被作为值存起来。使用选择器指代一个方法但不去调用它，就可以拿到这个方法的值，将其赋给一个变量，就可以在其他地方调用这个方法。赋给变量后，这个方法的receiver参数已经被确定，传入剩下的非receiver参数即可。

```go
p := Point{1, 2}
q := Point{4, 6}
distanceFromP := p.Distance // method value
fmt.Println(distanceFromP(q)) // "5"
var origin Point // {0, 0}
fmt.Println(distanceFromP(origin)) // "2.23606797749979", ;5
scaleP := p.ScaleBy // method value
scaleP(2) // p becomes (2, 4)
scaleP(3) // then (6, 12)
scaleP(10) // then (60, 120)
```

方法值的一个应用是作为回调函数使用，它本质就是一个绑定了receiver的普通函数。当一个方法除了receiver之外没有其他参数，这个方法值独立出来就是对这个特定receiver的某种确定的操作。例如下面的`r.Launch()`。

```go
type Rocket struct { /* ... */ }
func (r *Rocket) Launch() { /* ... */ }
r := new(Rocket)
time.AfterFunc(10 * time.Second, r.Launch)
```

### 方法表达式

我们还可以直接通过类型名去访问它上面的一个方法，进而获取到一个方法**本身**。如果方法的接收器是指针类型，就从指针类型上获取：`(*Point).Distance`。这种访问一个类型上方法的表示称为方法表达式（method expression）。
```go
distance := Point.Distance
distance(p, q)
```

通过方法表达式拿到的方法（或者直接称为函数）的第一个参数固定为receiver。在这里就没有语法糖了，必须按照原本的类型传入。有了拿到方法本身的方法，我们可以对方法进行条件性的动态调用。

```go
type Point struct{ X, Y float64 }
func (p Point) Add(q Point) Point { return Point{p.X + q.X, p.Y + q.Y} }
func (p Point) Sub(q Point) Point { return Point{p.X - q.X, p.Y - q.Y} }
type Path []Point

func (path Path) TranslateBy(offset Point, add bool) {
    var op func(p, q Point) Point
    if add {
        op = Point.Add
    } else {
        op = Point.Sub
    }
    for i := range path {
        // Call either path[i].Add(offset) or path[i].Sub(offset).
        path[i] = op(path[i], offset)
    }
}
```

看到这里，就可以发现Go在function-as-value方面的灵活性真不少，虽然仍比不上JavaScript（以及C++？）里的各种“魔法”。
