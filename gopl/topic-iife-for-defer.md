# 主题：为了用`defer`去使用IIFE

`defer`可以确保在一个函数返回的时候，相关函数一定被执行，很适合用来做一些cleanup。但是在一般的循环中却做不到这一点，因为循环只是函数体中的一个普通的结构。

如果这个循环很大，或者永远不会停止呢？例如，系统中的一些常驻任务就是一个有时间间隔、无条件的for循环，在一个单独的goroutine中执行。这个时候我们可以考虑将每一轮循环的逻辑封装成一个函数去调用。但有的时候我们可能希望代码更加紧凑，或者这个函数本身依赖一些外部的状态，而这些状态是由包围函数的局部变量定义的，此时就不能将它分离出去。

例如在一个不算长的逻辑中引入了上下文，并且希望每一轮循环执行结束之后都能将上下文取消掉，如果要手动编写的话比较麻烦且不够优雅，就考虑到使用defer。在这里我们可以不用分离这部分逻辑，而是直接将逻辑用一个`func () {...}`包围起来，构成一个IIFE，从而让这段逻辑更紧凑（保留在循环体内，而不是转移出去）。

```go
// ...
for {
    select {
        case <-ticker.C:
        // ...
        func () {
            ctx, cancel := context.WithCancel(context.Background())
            defer cancel()
            // ...
        }()
        // ...
    }
}
```

:::tip What is IIFE?
IIFE是指Immediately Invoked Function Expression，即立即调用的函数表达式。这种叫法一般只用在JS里面，因为确实有很多JS magic需要通过它来实现。为了简单，我把Go中的这种定义函数字面量以后马上执行的写法也叫做IIFE。其实任何有类似的函数式特性（函数作为值使用）的语言都应该存在IIFE。
:::

IIFE创建了一个新的block/作用域，其内部的标识符在外部不可访问，并且存在遮蔽行为，需要注意。
