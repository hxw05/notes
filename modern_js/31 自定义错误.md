# 31 自定义错误

JS中，由于throw关键字后面可以跟任何类型的值，所以我们完全可以使用自己定义的对象来表示一个错误。但是为了方便检测，最好继承自Error。

```jsx
class MyError extends Error {
  constructor(message) {
    super(message);
    this.name = this.constructor.name; // 自动设置为类名（构造器名），不用手动设置
  }
}

class ValidationError extends MyError { }

class PropertyRequiredError extends ValidationError {
  constructor(property) {
    super("No property: " + property);
    this.property = property;
  }
}
```

## 异常包装

可以将同类的Error包装成同一个Error来简化检查。例如，可以将JSON解析过程出现的SyntaxError和ValidationError等包装成一个ReadError，在catch中做检查时就只需要instanceof ReadError。

可将原始Error的引用存储到ReadError上的一个字段cause里。
