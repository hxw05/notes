# 07 对象

## 创建对象

```jsx
let user = new Object(); // constructor
let user = {}; // literal
```

## 对象属性名

对象的属性名总是字符串。当使用其它数据类型时，将尝试将其转换为字符串。

```jsx
a = {};
a[{}] = 123;
console.assert(a['[object Object']) === 123);
```

对象的__proto__指针的值永远是一个对象，将其设置为非对象是no-op。

## in操作符

in操作符用来判断一个键是否存在于对象中。

## for … in

for … in 用于遍历一个对象的所有键。一般遍历的顺序是这些键被创建的顺序：

```jsx
let abc = {a: 1, b: 2}
abc.d=3; abc.c=4;
for (let k in abc) { console.log(k); }
// a b d c
```

存在一种特殊情况，当键名为纯整数（不带任何非数字符号的整数）时，这些键会被自动排序为升序。例如

```jsx
let codes = {
	"def": "def",
  "49": "Germany",
  "41": "Switzerland",
  "44": "Great Britain",
  // ..,
  "1": "USA",
  "abc": "abc"
};

for (let code in codes) {
  alert(code); // 1, 41, 44, 49, def, abc
}
```

## 变量总是存储对象的引用

对象创建出来就是一个引用，例如

```jsx
let a = {};
let b = a;
```

a和b都是指向同一个对象的引用。这种情况下，满足 a==b 和 a===b。

## 浅拷贝

假设对象中的值都是基本类型，则可以实现简单的浅拷贝。例如：

```jsx
let obj = { /* ... */ };
let obj2 = {};

for (let k in obj) {
	obj2[k] = obj[k];
}
```

或者使用Object.assign，第一个参数为目标对象，其余为来源对象。相当于全部he'b到第一个对象中，有同键名则后一个覆盖前一个。

```jsx
let user = { name: "John" };

let permissions1 = { canView: true };
let permissions2 = { canEdit: true };

// 将 permissions1 和 permissions2 中的所有属性都拷贝到 user 中
Object.assign(user, permissions1, permissions2);
```

或者用spread。

JSON.parse(JSON.stringify())也是一种浅拷贝的方式，可以一定程度上实现浅拷贝。缺点：无法拷贝对象中的函数等，无法处理引用循环（stringify无法序列化带循环的引用）。

## 深拷贝

深拷贝可以看成是递归的浅拷贝。先拷贝最顶层的所有属性，对于基本类型则直接赋值，遇到非基本类型（数组、对象等，都是object），就对它们也应用一层拷贝，逐层递归。

通常使用现成的深拷贝实现，如lodash的cloneDeep。

## this

this的一个比较直观的用法是挂在对象上的函数访问对象内部的值。

```jsx
const obj = {
	a: 1,
	getA() {
		return this.a;
	}
}
```

getA函数也可以脱离对象，因为this的值是运行时才确定的。

```jsx
function getA() {
	return this.a;
}

const obj1 = {
	a: 1,
	getA,
};

const obj2 = {
	a: 2,
	getA,
}

console.assert(obj1.getA() === 1);
console.assert(obj2.getA() === 2);
```

如果直接调用getA，会出错，因为当这样一个函数没有挂在任何对象上的时候，this的值是undefined。这是严格模式下的固定行为，其在 [01 Strict Mode](./01 Strict Mode.md) 有所提及，这样做是为了让JS更安全。sloppy下，这样执行this会变为globalThis，浏览器里就是window。

箭头函数没有this。在箭头函数里面获取的this与其所处的环境有关。

## 对象到原始值的转换

对象之间发生运算时，需要转换为原始值，而不是产生新的对象。

对象作为布尔值时总是true，不讨论。

对象的类型转换有三种

- string，发生在明确需要字符串的上下文，如作为键名
- number，发生在明确需要数字的上下文，如正号后面。另外，由于历史原因，虽然大于号和小于号左右可以是字符串也可以是数字，但对于对象而言，总是使用number类型转换。
- default，发生在不清楚的上下文，其中一种情形是进行不严格比较（==）的时候，此外还有如加号，因为加号可以连接字符串，也可以做加法。

对象转换的过程：

1. 尝试调用对象上的一个隐藏方法，对应的symbol为Symbol.toPrimitive。该方法接受一个参数hint（string/number/default）。
2. 如果不存在，若hint=string，则尝试调用toString()和valueOf()；若hint=number/default，则尝试调用valueOf()和toString()

### Symbol.toPrimitive

```jsx
let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    alert(`hint: ${hint}`);
    return hint == "string" ? `{name: "${this.name}"}` : this.money;
  }
};
```

像这样，就可以看作是“实现了”toPrimitive的功能。当hint===string时，这个函数所完成的任务类似于Python中的__repr__。

### toString, valueOf

toString和valueOf是很早就存在的方法，这两个方法总是返回基本类型。

默认情况下，对象的toString总是返回[object Object]，valueOf则返回对象本身【远古js的行为，可以当做valueOf不存在】

```jsx
alert(user); // [object Object]
alert(user.valueOf() === user); // true
```

我们也可以自己实现这两个方法。一般考虑实现toString方法，当Symbol.toPrimitive和valueOf都不存在的时候，toString会负责所有的基本类型转换。

toString和valueOf方法应当返回基本类型，但是如果返回对象也不会报错，而是会被忽略掉。toString方法不一定要返回string（但可能会让这个函数变得confusing）。

Symbol.toPrimitive如果不返回基本类型将出现错误。
