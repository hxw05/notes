# 10 原始类型补充：方法

原始类型虽然表示的是简单值，但是在这种值上依然存在属性和方法。

## 包装类型

为了实现这一点，JS为原始类型提供了相应的包装类型，如number对应Number，bigint对应BigInt，symbol对应Symbol等。

通常不应该使用包装类型构造函数，如 new Number，这样将得到一个object。但在不带new的时候可以用作转型用途。

null和undefined不提供方法，也就没有包装类型。

## 访问过程

为了能够访问这些属性和方法，在调用时，会先创建一个对应原始值的包装对象实例，调用方法后再将实例销毁。这个过程是轻量、高度优化的。

## eg. 在一个原始值上设置property

```jsx
let str = "Hello";

str.test = 5; // (*)

alert(str.test);
```

由于这里访问了property，因此(*)行一个包装类型**会被创建**。但是，这样的操作是一种no-op，原始类型不能当成对象来使用，不能存储除了原始值以外的额外数据。
