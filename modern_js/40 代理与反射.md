# 40 代理与反射

## 代理对象

```jsx
let proxy = new Proxy(target, configuration);
```

使用new Proxy可创建target的一个代理对象，其中configuration为代理相关配置，{}表示空的代理配置。代理对象可以拦截对target的一系列操作，添加额外的逻辑，并选择是否将这些操作转发到target上。

代理对象是一种奇异对象，它不具有任何自己的属性（own property）。

### 对象属性访问内部方法

在JS标准中，对对象属性各种类型的访问均由特殊的内部方法实现，例如读取对象属性值的内部方法为[[Get]]，设置对象属性值的内部方法为[[Set]]等。Proxy通过拦截这些内部方法来达到拦截针对对象的操作的效果。

- 这些内部方法与Proxy配置上的陷阱函数（trap）名称有如下对应关系：
    
    
    | 内部方法 | Handler方法名称 | 触发时机 |
    | --- | --- | --- |
    | [[Get]] | get | 读取属性 |
    | [[Set]] | set | 写入属性 |
    | [[HasProperty]] | has | in操作符 |
    | [[Delete]] | deleteProperty | delete操作符 |
    | [[Call]] | apply | 函数调用 |
    | [[Construct]] | construct | new操作符 |
    | [[GetPrototypeOf]] | getPrototypeOf | [Object.getPrototypeOf](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/getPrototypeOf) |
    | [[SetPrototypeOf]] | setPrototypeOf | [Object.setPrototypeOf](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/setPrototypeOf) |
    | [[IsExtensible]] | isExtensible | [Object.isExtensible](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/isExtensible) |
    | [[PreventExtensions]] | preventExtensions | [Object.preventExtensions](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/preventExtensions) |
    | [[DefineOwnProperty]] | defineProperty | [Object.defineProperty](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty), [Object.defineProperties](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperties) |
    | [[GetOwnProperty]] | getOwnPropertyDescriptor | [Object.getOwnPropertyDescriptor](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/getOwnPropertyDescriptor), for..in, Object.keys/values/entries |
    | [[OwnPropertyKeys]] | ownKeys | [Object.getOwnPropertyNames](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/getOwnPropertyNames), [Object.getOwnPropertySymbols](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/getOwnPropertySymbols), for..in, Object.keys/values/entries |

每一个陷阱函数上都有一些约束，实现这些陷阱函数时应当满足约束，从而维持一致的语言行为。这些约束例如

- set、delete、…操作必须返回一个布尔值，表示操作成功与否
- getPrototypeOf必须返回target的原型

### 实例

#### 1. 使用get陷阱函数实现默认值

- get(target, property, receiver) 通常用到前两个参数，target是原对象，property是尝试获取的字段名。

```jsx
let numbers = [0, 1, 2];

numbers = new Proxy(numbers, {
  get(target, prop) {
    if (prop in target) {
      return target[prop];
    } else {
      return 0; // default value
    }
  }
});
```

:::tip
代理对象的最佳实践是用代理对象替换掉原对象变量 numbers=new Proxy(numbers, …)，从而防止对原对象的访问。

一个对象一旦被代理，其原对象就成了某种意义上的内部对象，如果能被外界访问可能造成意料之外的效果。
:::

#### 2. 使用set陷阱函数检查值的合法性

- set(target, property, value, receiver)

假设要实现一个纯数字数组，该数组中只允许添加number值。

```jsx
let numbers = [];

numbers = new Proxy(numbers, { // (*)
  set(target, prop, val) { // to intercept property writing
    if (typeof val == 'number') {
      target[prop] = val;
      return true;
    } else {
      return false;
    }
  }
});

numbers.push(1); // added successfully
numbers.push(2); // added successfully
alert("Length is: " + numbers.length); // 2

numbers.push("test"); // TypeError ('set' on proxy returned false)

alert("This line is never reached (error in the line above)");
```

注意此处我们只是实现了对set操作的拦截，而没有修改Array.prototype上的函数，但它们（push、unshift等方法）仍然按预期工作了。这是因为他们内部使用到了set操作。

set方法返回true表示写入成功，返回false则相当于抛出一个TypeError。也可以直接throw

#### 3. 使用ownKey陷阱函数和getOwnPropertyDescriptor陷阱函数自定义迭代行为

- ownKeys(target)，总是返回一个字符串数组，表示target上的所有自身键

会访问到这些陷阱函数的操作包括

- Object.keys，返回可列自身非symbol键
- Object.values，返回可列自身非symbol键的值
- Object.getOwnPropertyNames，返回所有自身非symbol键
- Object.getOwnPropertySymbols，返回所有自身symbol键
- for … in 遍历所有可列非symbol键，包含原型键

假设我们希望隐藏一个对象上所有以_开头的属性，使其无法被遍历，我们可以让ownKeys返回一个可列自身非symbol键的子集，通过对Object.keys(target)进行filter

```jsx
let user = {
  name: "John",
  age: 30,
  _password: "***"
};

user = new Proxy(user, {
  ownKeys(target) {
    return Object.keys(target).filter(key => !key.startsWith('_'));
  }
});

// "ownKeys" filters out _password
for(let key in user) alert(key); // name, then: age

// same effect on these methods:
alert( Object.keys(user) ); // name,age
alert( Object.values(user) ); // John,30
```

- getOwnPropertyDescriptor(target, property) 返回property字段的配置对象（descriptor）

如果我们让ownKeys返回原对象上不存在的键，那么此时再用Object.keys获取原对象的所有键时，**并不能**看到这些不存在的键。

```jsx
let user = { };

user = new Proxy(user, {
  ownKeys(target) {
    return ['a', 'b', 'c'];
  }
});

alert( Object.keys(user) ); // <empty>
```

这是因为Object.keys需要确保这些对象是自身拥有的、可列的，因此内部相当于会对每一个键都调用getOwnPropertyDescriptor，一是验证是否是own property，二是看配置对象上的enumerable是否是true。此处我们并没有拦截getOwnPropertyDescriptor，仍然是默认行为：看不到我们额外加上去的键。

解决方法是让getOwnPropertyDescriptor返回预期的字段配置即可。例如下面的例子中，就对任意prop值的getOwnPropertyDescriptor调用返回了可列、可配置（enumerable: true，configurable: ture）信息。

```jsx
let user = { };

user = new Proxy(user, {
  ownKeys(target) { // called once to get a list of properties
    return ['a', 'b', 'c'];
  },

  getOwnPropertyDescriptor(target, prop) { // called for every property
    return {
      enumerable: true,
      configurable: true
      /* ...other flags, probable "value:..." */
    };
  }

});

alert( Object.keys(user) ); // a, b, c
```

#### 4. 综合使用陷阱函数实现对_开头字段的保护

- deleteProperty(target, property) 返回一个布尔值，删除成功为true

```jsx
let user = {
  name: "John",
  _password: "***"
};

user = new Proxy(user, {
  get(target, prop) {
    if (prop.startsWith('_')) {
      throw new Error("Access denied");
    }
    let value = target[prop];
    return (typeof value === 'function') ? value.bind(target) : value; // (*)
  },
  set(target, prop, val) { // to intercept property writing
    if (prop.startsWith('_')) {
      throw new Error("Access denied");
    } else {
      target[prop] = val;
      return true;
    }
  },
  deleteProperty(target, prop) { // to intercept property deletion
    if (prop.startsWith('_')) {
      throw new Error("Access denied");
    } else {
      delete target[prop];
      return true;
    }
  },
  ownKeys(target) { // to intercept property list
    return Object.keys(target).filter(key => !key.startsWith('_'));
  }
});
```

注意get中返回的`(typeof value === 'function') ? value.bind(target) : value;`，主要是出于对字段值为函数，函数中访问了target（通过this）上的隐藏属性的情况的考虑，一般应当允许对象上的函数访问自身的任何属性，即使这些属性是隐藏的。

这其实是一个不甚完备的处理，需要更强的约定才能良好工作：函数的行为很多，如果将原对象target到处返回、传值等，很可能会造成混乱，也无法起到保护私有属性的作用。

#### 5. 使用has陷阱函数自定义in操作符的行为

- has(target, property) 在使用in操作符时被调用，如果property存在则返回true

```jsx
let range = {
  start: 1,
  end: 10
};

range = new Proxy(range, {
  has(target, prop) {
    return prop >= target.start && prop <= target.end;
  }
});

alert(5 in range); // true
alert(50 in range); // false
```

#### 6. 使用apply陷阱函数来包装其他函数

- apply(target, thisArg, args) thisArg是调用时的this，args是参数（数组）

在使用装饰器模式时，delay返回的是包装以后的函数（function () {…）。我们无法对原函数进行一些操作，或者访问一些属性，例如length。

```jsx
function delay(f, ms) {
  return function() { // (*)
    setTimeout(() => f.apply(this, arguments), ms);
  };
}

function sayHi(user) {
  alert(`Hello, ${user}!`);
}

sayHi = delay(sayHi, 3000);
console.log(sayHi.length); // 0 而不是 1
```

可以使用代理对象来实现一样的效果

```jsx
function delay(f, ms) {
	return new Proxy(f, {
		apply(target, thisArg, args) {
			setTimeout(() => {
				target.apply(thisArg, args)
			}, ms);
		}
	})
}
```

## 反射

在JS中，反射（Reflect）是对一些关键字操作/内部方法的函数调用，代理中可以配置的任意一个陷阱函数，都有一个反射函数与之对应，其参数含义和顺序完全相同。

反射的一个重要作用是将代理对象上的操作按需转发给原对象。虽然反射函数本来就有所对应的语言语法，但是很多时候语法并不能准确表达此处发生的事情，最简单的仍然是将代理中的陷阱函数的参数原封不动地传递给反射函数。

### 实例：为什么应该使用反射函数

下面的例子中，我们对user进行了代理，假设get陷阱函数中所有其他操作都被省略，我们专门考察最后的转发环节是否能够正常进行。在这里我们使用了语言语法自带的get表达：使用中括号直接访问原始对象上的某个属性。

```jsx
let user = {
  _name: "Guest",
  get name() {
    return this._name;
  }
};

let userProxy = new Proxy(user, {
  get(target, prop, receiver) {
    return target[prop];
  }
});

let admin = {
  __proto__: userProxy,
  _name: "Admin"
};

// Expected: Admin
alert(admin.name); // outputs: Guest (?!?)
```

此处admin.name访问过程：

- 是读操作，在admin上面找不到name字段，于是开始在原型链上寻找
- 原型链向上走一级到userProxy上，找到了name字段；该字段是一个getter，所以对其进行调用
- getter内部返回this._name（注意此时this是admin），发生了get操作，被代理上的get陷阱函数拦截
- get陷阱函数返回target[prop]，其中target是user，prop是_name，于是就获取到了不正确的Guest

需要注意的是，getter写法是没有错误的，如果不是继承的代理对象，getter中的this绑定的是admin，获取到的应该就是admin._name；而由于代理对其进行了拦截，导致了行为的改变。

get陷阱函数的第三个参数receiver正是用于解决这种情形，在此处receiver=admin，但是我们没有使用它。将return target[prop]修改为Reflect.get(target, prop, receiver)即可解决问题，或者Reflect.get(…arguments)也可以。

## 代理的限制

### 绕过get/set陷阱函数的内部数据管理机制

Proxy不一定能够达到想象中的效果，这主要是因为绕过get/set陷阱函数的内部数据管理机制。

JS提供的一些数据结构，例如Map，内部并不使用[[Get]]或者[[Set]]去读取和写入数据，而是维护一套内部的数据存放点（称为internal slots），在Map上可以用[[MapData]]来代指。这就导致Proxy无法对其进行拦截，也就无法达到预想的效果。

在下面这个例子中，为map创建了一个透明代理。设想中，这个proxy的表现应该与map相同，但调用set方法时出现错误。

```jsx
let map = new Map();

let proxy = new Proxy(map, {});

proxy.set('test', 1); // 出错
```

这主要是因为调用set时，其this指针被替换为了代理对象而不是原本的对象，而代理对象上面不存在这种internal slot（无法访问this.[[MapData]]）。

一种workaround是特殊对待这些函数调用，如果发现是函数，就将this替换为原始对象。

```jsx
let map = new Map();

let proxy = new Proxy(map, {
  get(target, prop, receiver) {
    let value = Reflect.get(...arguments);
    return typeof value == 'function' ? value.bind(target) : value;
  }
});

proxy.set('test', 1);
alert(proxy.get('test')); // 1 (works!)
```

这种替换的方式带来的最大的问题是，它仍然将被代理的原始对象暴露在外，使得方法可以自由地对原始对象做操作以及传递，很可能破坏代理的功能。

:::tip
数组没有internal slot，因此数组的代理是完全可用的。
:::

除了Map等数据结构可能使用内部的数据管理机制外，类的私有字段也使用了内部的数据管理机制，而没有使用[[Get]]/[[Set]]操作。

在下面的例子中，代理上调用getName时，this是代理而不是原始对象，而代理上并不存在私有字段相关数据，导致出错。解决方法与先前类似，仍然是拦截时针对方法调用特殊处理，使其this仍然为原始对象。

```jsx
class User {
  #name = "Guest";

  getName() {
    return this.#name;
  }
}

let user = new User();

user = new Proxy(user, {});

alert(user.getName()); // Error
```

### 代理不支持修改等值判断行为

- Proxy不能拦截一个严格的等值判断 `===`
- 代理对象不等于原始对象；如果一个Set、数组等结构中存有原始对象，那么includes、has针对代理对象均返回false。

## 可撤销代理

可撤销代理可以通过Proxy.revocable方法来创建，其参数与new Proxy相同。

```jsx
let {proxy, revoke} = Proxy.revocable(target, handler)
```

一个主要的用法是创建一个透明的可撤销代理，就相当于是对一个对象的有限制的访问，可以根据需要随时撤销掉。

```jsx
let object = {
  data: "Valuable data"
};

let {proxy, revoke} = Proxy.revocable(object, {});

alert(proxy.data); // Valuable data

revoke();

alert(proxy.data); // Error
```

为了方便找到revoke方法，可以使用WeakMap。这样当作为键的proxy被回收后，revoke的可达性也不会阻止GC，会一并回收掉。

## Examples

### 1. 实现负下标

需要注意prop是一个string。

```jsx
let array = [1, 2, 3];

array = new Proxy(array, {
  get(target, prop, receiver) {
      prop = +prop;
      if (isNaN(prop)) return undefined;
      if (prop < 0) {
          let newIndex = target.length + prop;
          if (newIndex < 0) return undefined;
          return Reflect.get(target, newIndex, receiver);
      }
      return Reflect.get(target, prop, receiver);
  }
});
```

### 2. makeObservable为对象添加一个observe方法（要求返回Proxy），实现传入回调 (key, value) => …

```jsx
function makeObservable(target) {
    let callback; // 使用闭包来存储回调函数
	  return new Proxy(target, {
	      get(tg, prop, receiver) {
	          if (prop === 'observe') {
	              return (cb) => callback = cb;
	          }
	          return Reflect.get(tg, prop, receiver);
	      },
	      set(tg, prop, value, receiver) {
	          callback?.(prop, value);
	          return Reflect.set(tg, prop, value, receiver);
	      }
	  });
}
```
