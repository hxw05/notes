# 37 异步迭代

在一个对象上面附加一个Symbol.iterator并使之返回一个带有next函数的对象（或者生成器实例），next函数每次调用返回{value, done}格式数据，就为这个对象定义了一个普通的迭代器。

Symbol.asyncIterator是另一种迭代器：异步迭代器，这个函数也需要返回一个带有next函数的对象，next函数每次调用返回一个Promise<{value, done}>；可以用async来定义next函数。

注意：Symbol.asyncIterator的用途有限，不可与Symbol.iterator通用。诸如普通的for…of和spread语法均只接受Symbol.iterator。

## 异步迭代

要对一个对象进行异步迭代，使用 `for await (let item of iterable)` 语法。iterable必须实现了Symbol.asyncIterator。

## async function*

与Symbol.iterator可以直接用一个生成器函数表示类似，一个Symbol.asyncIterator也可以简化为一个async function*，对该迭代器实例调用的每一次next都会返回promise而非值，使得其表现与Symbol.asyncIterator相符。

下面是iterable range的异步迭代实现

```jsx
const range = {
	from: 5,
	to: 10,
	async *[Symbol.asyncIterator]() {
		for (let i = this.from; i <= this.to; i++) {
			// 这里可以加一点await什么的
			yield i;
		}
	}
}

for await (let i of range) {
	console.log(i); // 5,6,7,8,9,10
}

```

## 实例：使用异步迭代实现exhaustive load/pagination

有如下流程，通过listV2方法，将每个分页下的res.objects都放到result中，实现数据的完整获取。使用异步生成器可以将下面的流程封装成函数。

```jsx
let continuationToken = '';
const result = [];

while (true) {
    let res = await client.listV2({
        "max-keys": 1000,
        "continuation-token": continuationToken
    });
    
    result.push(...res.objects);

    if (res.isTruncated) {
        continuationToken = res.nextContinuationToken;
    } else {
        continuationToken = '';
        break;
    }
}
```

封装成异步生成器，屏蔽内部细节。

```jsx
async function* listV2Exhaustive() {
	let continuationToken = '';
	
	while (true) {
	    let res = await client.listV2({
	        "max-keys": 1000,
	        "continuation-token": continuationToken
	    });
	    
	    for (let object of res.objects) {
		    yield object; // *
	    }
	
	    if (res.isTruncated) {
	        continuationToken = res.nextContinuationToken;
	    } else {
	        continuationToken = '';
	        break;
	    }
	}
}
```

这样在外部可以直接使用listV2Exhaustive来获取到所有object。

```jsx
const objects = [];

for await (let object of listV2Exhaustive()) {
	objects.push(object);
}
```

这样看上去就简洁了不少。
