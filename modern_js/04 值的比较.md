# 04 值的比较

## 字符串的比较

字符串的比较实质是在比较两个字符串中对应位置字符的Unicode编码顺序，遵循如下原则：

- 从两个字符串的开头，一对一比较
- 如果某个位置能够分出大小，就是这两个字符串的大小
- 如果到最后都分不出大小，其大小取决于字符串的长度（长度相等、较长、较短等）。

## 不同类型之间的比较

不同类型之间的大小比较和不严格的等值比较，总是先转换为数字再进行比较。因此，在研究这些结果的时候常常需要记住不同值转换为数字的值是什么，比如

- 空字符串→0
- null→0，在等值中特殊
- undefined→NaN
- …

```jsx
console.assert('2' > 1)
console.assert('2' > '1')
console.assert('01' == 1)
console.assert(null >= 0)
console.assert(undefined != 0)
```

### 不严格相等

普通的==属于不严格比较的一种，它会将两端先转换为数字再进行比较。

```jsx
console.assert(0 == false)
console.assert(0 == '')
```

但是也有例外：

- 0==null 返回的是false。虽然null可以被转换为0，但是此行为在loose equality却不存在（而在≥、≤中均存在，如null≥0 === true）。规定：在不严格等值性中，只有null==undefined是成立的（当然null==null和undefined==undefined是成立的）。
- 这也是为什么0==undefined也是false（并不是因为undefined被转换为NaN，虽然看起来也合理。但归根结底还是存在仅null==undefined的规定）
