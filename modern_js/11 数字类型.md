# 11 数字类型

## toString(base)方法

- 如果要在一个不带小数部分的number literal上调用方法，需要使用两个点号，如 1234..toString()

toString方法返回数字给定的进制表示，base的范围为2~36，默认为10。

## 小数Math方法

- Math.floor，Math.ceil
- Math.round：最近的整数，中间值向上（-3.5 → -3，3.5 → 4）
- Math.trunc：移除小数点，不舍入

## 精度问题, toFixed

JS中的number本质上是64位双精度浮点型，无法精确表示0.1和0.2。在必要的时候，需要进行舍入来舍弃那些非常小但是会导致不相等的误差。

不仅是小数，整数也会有不精确，例如9999999999999999的运算结果为10000000000000000。

### toFixed

- 对于负数的行为与Math.round不同
- 该函数很大程度上受到精度问题的影响。

toFixed(n)保留到小数点后n位。注意+num.toFixed(0)与Math.round(num)不等价。对于负数，toFixed相当于将其看成正数进行舍入，然后再将符号加回去。

```jsx
const num = -3.5;
console.assert(+(num).toFixed(0) === -4);
console.assert(Math.round(num) === -3);
```

toFixed基于某个数字的表示值。如果某个数字无法被精确表示，toFixed结果也会受到影响，如

```jsx
6.35.toFixed(20)
// => '6.34999999999999964473'
```

这就导致toFixed(1)的结果为6.3而非预期的6.4。为了解决，需要让其更接近整数，例如保留1位小数时，乘以10并使用Math.round进行舍入，最后再除回去。

```jsx
Math.round(6.35*10)/10
// => 6.4
```

### 其它精度问题

由于精度问题是不可解决的，因此需要避免。例如，循环结束条件不应该依赖于小数的等值判断，否则可能导致意料之外的无限循环。

## isFinite, isNaN

- isNaN是用于判断一个值是否为NaN的方法（但不是唯一的方法）
- isFinite用于判断是否为一个有效的数字，即不是NaN、Infinity和-Infinity。该函数会将其参数自动转换为数字进行比较。注意由于空字符串转换为数字为0，因此isFinite(’’)===true。

### Object.is

Object.is也可以用来判断一个值是否为NaN，例如

```jsx
console.assert(Object.is(NaN, NaN));
console.assert(Object.is(+'12a', NaN));
```

0和-0两个值一般被看作是相同的，在运算中他们没区别，不过Object.is可以区分它们。

除了以上两种情况外，Object.is和===的作用相同。

## parseInt, parseFloat

这两个函数的主要作用是不严格地将字符串转换为数字。

从字符串的开头开始读取，直到结束或者某个字符不再符合数字的格式。

```jsx
console.assert(parseInt('12%') === 12)
console.assert(parseInt('1.12.5') === 1)
console.assert(parseInt('0xff', 16) === 255)
console.assert(parseInt('ff', 16) === 255)
console.assert(parseFloat('1.12.5') === 1.12)
```

parseInt的第二个参数是radix，可以指定解析数字系统的基数，范围仍然为2~36，如果超出返回则返回NaN。

## 其它数学函数

- Math.random 返回一个 $[0,1)$ 范围内的随机数。
    - 范围随机数：min + Math.random() * (max-min)， $[0,1)\rightarrow [0,M-m)\rightarrow [m, M)$
    - 均匀的范围随机整数（生成每个整数的概率相等），包含两端：
        - min - 0.5 + Math.random() * (max - min + 1), Math.round
            
            随机数范围为 $[m-0.5, M+0.5)$，使用round后的范围为 $[m, M+1) \cap \Z$ 也就是 $[m,M]\cap\Z$
            
            并且：
            
            - m-0.5..m+0.49999 → m
            - m+0.5..m+1+0.49999 → m+1
            - m+1+0.5..m+2+0.49999 → m+2
            - …
            - M-0.5…M+0.4999 → M
            
            由于每一段的长度都是相等的，因此是等概率的。
            
        - min + Math.random() * (max - min + 1), Math.floor
            
            随机数范围为 $[m,M+1)$，使用floor后的范围为 $[m, M]\cap\Z$
            
            并且：
            
            - m..m+0.99999 → m
            - m+1..m+1+0.99999 → m+1
            - …
            - M..M+0.99999 → M
            
            由于每一段长度都是相等的，因此是等概率的。
            
            如果生成的随机数范围没有跨越0，则Math.floor可以替换为Math.trunc，否则将导致取到0的概率比取到其他任何整数的概率均大一倍，无法实现等概率。（因为Math.trunc对于正数等价Math.floor，而对于负数等价Math.ceil，取到0的区间为 [-0.9999, 0.9999]（长度~=2），而其它任意区间满足 [m, m+0.9999]（长度为~=1）。
            
        - （与上一个同理） min-1+Math.random() * (max - min), Math.ceil
            
            
- Math.max(…args), Math.min(…args)
- Math.pow
