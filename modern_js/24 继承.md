# 24 继承

与对象的继承类似，类的继承也是使用原型链来实现的。

## Rabbit extends Animal

- 将Rabbit.prototype的原型指针（__proto__）指向Animal.prototype
- 这样，当访问Rabbit实例上没有的属性或方法时，就会沿着原型链在Animal上查找

## super

用于调用父类的构造函数或方法。

- super(…) 调用构造函数，只能在constructor中调用
- super.method(…) 调用方法

箭头函数没有super，其super需要在作用域中捕获使用。当用作回调函数时，应传入箭头函数而非普通函数，因为普通函数会尝试动态地绑定super，但由于缺少类上下文，会绑定失败。

## 子类的constructor

Note: 为了方便，将派生类简称为子类。

在规范中，如果一个子类没有构造函数，则采用一个默认的构造函数。该构造函数相当于父类的构造函数。

```jsx
constructor(...args) {
	super(...args);
}
```

如果子类有构造函数，需要遵循下面的规则

- 必须调用super
- this赋值需要出现在super之后

### 派生构造器

派生构造函数内部会有一个特殊的属性标记[[ConstructorKind]]: derived

一般的构造函数，new之后会新建一个空对象作为this以及最终返回的实例对象。而标有derived的构造函数没有新建空对象这一步。它需要super来完成这个过程。因此必须先调用super，再操作this。

### 类字段初始化顺序

类的字段的初始化的顺序在子类和基类之间有所不同。

- 一个基类的字段在构造函数执行之前初始化
- 一个子类的字段在构造函数中super调用之后立即初始化

尤其注意第二个顺序，这表示当new一个子类时，其父类构造函数在执行时将看不到子类的任何一个字段。如果子类有对父类字段的重载，父类构造函数super中仍然只能读取到自己存储的字段。

但是，getter/setter/方法不受此规则影响。

## 拓展：super的实现原理

### 自然但不正确的实现

一个比较直观的super实现，是用this.__proto__.parentMethod.call(this)。这里通过当前对象的proto指针来获取到父类的方法，并且为了正确的上下文（而不是让this变成prototype对象），使用call(this)来调用。

```jsx
let animal = {
  name: "Animal",
  eat() {
    alert(`${this.name} eats.`);
  }
};

let rabbit = {
  __proto__: animal,
  eat() {
    // ...bounce around rabbit-style and call parent (animal) method
    this.__proto__.eat.call(this); // (*)
  }
};

let longEar = {
  __proto__: rabbit,
  eat() {
    // ...do something with long ears and call parent (rabbit) method
    this.__proto__.eat.call(this); // (**)
  }
};
```

这样做会导致问题：

- 当longEar的eat被调用时，this.__proto__可以正确找到父类对象rabbit。用longEar作为this来调用rabbit.eat。在这一步，super.eat()被正确实现了。
- 但是，由于longEar被作为rabbit.eat的this，在rabbit.eat中想要继续向上走一层时，却获取不到animal.eat，而是折返回了longEar.eat，导致了循环。longEar.eat变成了一个无限调用自己的函数。

这个问题是该实现方法的局限性所致，但除此以外也找不到更好的解决方式。

### [[HomeObject]]

单单依靠this无法实现super。

JavaScript为函数添加了HomeObject内部属性，这个属性的作用是标记一个方法原本来自于的那个对象。

当一个函数被用作类或者**对象**上的方法时，它的HomeObject值存储的是指向类实例或者对象的引用，即 longEar.eat.[[HomeObject]]===longEar。super通过HomeObject来寻找父类实例上的方法和解决循环的问题。

- 在方法被创建时，它的[[HomeObject]]就已经确定。此时如果传递其引用，[[HomeObject]]不会改变。这代表如果将一个使用了super的方法传递给另外一个对象，super的指向仍然指向原对象的父类对象。
- [[HomeObject]]仅用于super关键字的解析。
- [[HomeObject]]只会在类方法或者对象方法上被创建，对于作为对象的字段值的函数，则不会创建，i.e. `eat: function()` 。在eat: function()这种写法中，将由于[[HomeObject]]不存在而导致super不可使用。这是两种写法之间少有的区别。
