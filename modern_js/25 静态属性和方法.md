# 25 静态属性和方法

F可能的称呼：类对象，类构造函数值，类构造函数

静态方法和静态属性相当于直接在F上定义的方法和属性。这些方法和属性不能在实例上访问。

```jsx
class User {
  static staticMethod() {
    alert(this === User);
  }
}

User.staticMethod(); // true

// 相当于

User.staticMethod = function() {
  alert(this === User);
};
```

## 静态属性和方法的继承

静态属性和方法的继承仍然通过原型链实现。

- 动态属性和方法的继承通过实例的原型的原型指针实现
    - rabbit.__proto__ === Rabbit.prototype
    - Rabbit.prototype.__proto__ === Animal.prototype
- 静态属性和方法的继承通过F上的原型指针实现，其指向的仍然是一个F，而非原型
    - Rabbit.__proto__ === Animal

## `class Rabbit` vs. `class Rabbit extends Object`

结合静态属性和方法继承的实现，归纳extends关键字的具体效果：

- 设置F.prototype的__proto__为父类prototype
- 设置F.__proto__为父类F

|  | class Rabbit | class Rabbit extends Object |
| --- | --- | --- |
| rabbit.__proto__ | Rabbit | Rabbit |
| Rabbit.prototype.__proto__ | Object.prototype | Object.prototype |
| Rabbit.__proto__ | Function.prototype
注意此处是原型，而非Function | **Object** |

另外有Object.__proto__ === Function.prototype，因此即使没有extends Object，一个F上仍然存在call、bind、apply等函数通用方法和属性（原型链F.__proto__ === Object, Object.__proto__ === Function.prototype）
