# 38 模块化简介

在ES6之前，JS并没有语言级别的模块化功能。这促使社区发明了若干模块化系统，包括

- AMD：最古老的模块化系统，基于require.js
- CommonJS：适用于Node.js的模块化系统
- UMD：一种通用的模块化系统，同时兼容AMD和CommonJS。

现在JS语言自带的模块化系统已经迭代了许多年，在许多主流浏览器中受支持。

## 模块

在JS中，模块指的就是单个JS文件。在浏览器中，模块使用script标签上的type=module来定义。

### `<script type="module">`

定义一个内联的模块脚本。

- import语句导入的脚本会通过HTTP获取到，执行，并获得导出的内容
- 不支持裸模块路径（bare module）
- 每一个script type=module都有自己的顶级作用域，彼此不互通
- 同一个模块只会执行一次
    
    ```html
    <!-- the script my.js is fetched and executed only once -->
    <script type="module" src="my.js"></script>
    <script type="module" src="my.js"></script>
    ```
    
- 远程模块需要CORS头才能获取
- 总是defer的，也就是在文档加载完毕后才执行，并且保留顺序（仅对于script type=module之间）
    
    ```html
    <!-- 一定是defer执行，在文档加载结束之后，可以获取到button -->
    <script type="module">
      alert(typeof button); // object
    </script>
    
    <!-- 正常执行，会在文档加载完之前运行，就获取不到button -->
    <script>
      alert(typeof button); // undefined
    </script>
    
    <button id="button">Button</button>
    ```
    
- 可以是async的。对于非模块<script>，是不支持内联脚本写async的。对于一个async内联module，会将所有import的模块获取并执行，然后再执行，不会等待其他无关模块的执行或者文档的加载。
    
    ```html
    <!-- all dependencies are fetched (analytics.js), and the script runs -->
    <!-- doesn't wait for the document or other <script> tags -->
    <script async type="module">
      import {counter} from './analytics.js';
    
      counter.count();
    </script>
    ```
    

### `<script nomodule>`

<script nomodule>会在不支持模块的浏览器中以非模块内联脚本运行。

## import/export

import语句用来导入模块，可以直接跟模块名，也可以跟需要导入的内容等。

- 在一个页面中，第一个导入某模块的import语句会执行该模块，后续的import不会再重复执行
- 根据上一条，导出的对象在多个import之间是共享的引用，这表示在一个模块中修改该对象上的字段，其他的模块也能看到修改。

### import.meta

import.meta是一个特殊对象，用于获取与当前模块相关的信息。例如，在浏览器中，import.meta.url给出当前脚本的URL；如果是内联脚本，这个URL指向HTML。
