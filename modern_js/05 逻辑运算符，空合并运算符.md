# 05 逻辑运算符，空合并运算符

## 逻辑运算符

### 性质：转换、短路、原样返回

- 逻辑运算符会将其操作数全部转换为布尔值进行计算
- 是短路运算

当逻辑运算符构成的表达式放在if中时，由于if语句的性质，该表达式的值总是会被转换为布尔值。

当逻辑运算符构成的表达式作为值使用时，

- 或运算总是寻找第一个truthy值，与运算总是寻找第一个falsy值。
- 一旦找到这个值，那么就将**原样返回**而不会将其转换为布尔值。因此逻辑运算符构成的表达式的返回值类型是任意的，而非一定是布尔值。

例如在React中：

```jsx
return array.length && array.map(x => <div key={x}>{x}</div>)
```

以上模板的渲染结果可能会出现“0”，因为当array.length === 0时为falsy，短路运算中止并返回**原值0**，会被React渲染出来。如果将array.length改为array.length > 0，短路运算提前中止时返回的是布尔值false，而React将false视为空节点。

在处理表单的结果时，也可以借助此特性：

```jsx
let firstName = "";
let lastName = "";
let nickName = "SuperCoder";

alert( firstName || lastName || nickName || "Anonymous"); // SuperCoder
```

### 优先级

或的优先级低于与，总是先算与，再算或。

非的优先级高于与和或。

## 空合并运算符

空合并运算符通常用于仅空值敏感的默认值判定。如果一个值是空的，就给它一个默认值，否则就是其原值。

与此相对应的是零值默认值，使用 `||` 来实现，如果一个值是零值（js中falsy值）就给它一个默认值，比如空字符串和数字0也属于需要给默认值的情况。

??的优先级和||的优先级是**相同的**，且??不能在不加括号的情况下与||和&&混用，否则产生Syntax Error。

```jsx
let x = 1 && 2 ?? 3; // Syntax error
let x = (1 && 2) ?? 3; // 2
```
