# 13 数组

## at方法

at是一个新的方法，当at(n) n≥0时，与下标访问相同，n < 0 时相当于从末尾向前取。例如n=-1时表示倒数第一个字符。

## push/pop/shift/unshift

- push在末尾插入
- pop在末尾删除，并返回该元素。
- unshift在开头插入
- shift在开头删除，并返回该元素。

使用push/pop可以实现栈结构，shift/unshift可以实现队列结构。因此，js中的数组可以看成一个双端队列（deque）。

push和pop由于操作都在数组的末尾，因此运行比shift/unshift要快。shift/unshift总是有一个移动剩余元素的过程。

## 数组是特殊的对象

数组也是一种对象，也具有引用性质。

JS的实现中倾向于将数组中的数据存储在连续的区域内，并进行一些其它的优化使数组运行的非常快。

这种优化仅限于按顺序地使用数组。如果像使用常规对象一样使用数组，例如，在数组上附加属性、制造空洞等，那么其对应的优化就会被关闭。

## for … in/of

不建议使用for … in，因为还会遍历一些非数字的属性，用在数组的语义上并不明确。这尤其体现在一些array like对象上，如NodeList。

for … in 用于数组速度相对较慢，会造成瓶颈。

## length

length属性是**可写的**，如果将其减少到某个值，那么其后的数据会被截断且不可恢复。清空数组最简单的方法是令其length=0。

## 数组的toString

数组没有Symbol.toPrimitive，也没有valueOf。数组的toString效果类似于join(”,”)

## 删除数组元素，splice

delete可以用来删除某个位置的值，但不会进行自动的移动，数组的length不会改变。

一般使用splice方法。

- arr.splice(start, deleteCount?, …e?) 从start位置（可以是负数）开始，删除deleteCount个元素并插入元素…e，返回被删除元素组成的数组。如果deleteCount, …e不填，表示从start位置开始删除到数组的末尾。

所以删除数组的第i个元素可以splice(i, 1)

当deleteCount被设置为0时，splice只会进行插入，且这种插入发生在start之前：

```jsx
let arr = ["I", "study", "JavaScript"];
arr.splice(2, 0, "complex", "language");
alert( arr ); // "I", "study", "complex", "language", "JavaScript"
```

## 截取slice

- arr.slice(start?, end?) slice返回一个新的数组，为原数组中start → end（不含end）范围的内容。

```jsx
["t", "e", "s", "t"].slice(1, 3) // e,s
["t", "e", "s", "t"].slice(-2) // s,t
```

## concat, Symbol.isConcatSpreadable

concat函数接收数组元素或者数组元素的数组作为参数，均会以元素为单位连接到原数组上，返回一个新的数组。

```jsx
let arr = [1, 2];
arr.concat([3, 4], 5, 6) // 1,2,3,4,5,6
```

如果concat的参数中有对象，即使其有数字属性，也会作为对象连接，除非对象上有Symbol.isConcatSpreadable这个键并指定为true。

```jsx
let arr = [1, 2];

let arrayLike = {
  0: "something",
  1: "else",
  [Symbol.isConcatSpreadable]: true,
  length: 2
};

alert( arr.concat(arrayLike) ); // 1,2,something,else
```

## indexOf, lastIndexOf, includes

与字符串的相关方法类似，它们在查找的时候均使用严格相等进行比较。

indexOf不能正确处理NaN，而includes可以（比较新）

## find, findIndex, findLastIndex

find返回数组中满足条件的那个元素，findIndex则返回其下标。

## sort【就地】

sort对数组进行就地排序，并且会返回修改后的数组，通常忽略这样的返回值。

sort默认情况下将其元素**按照字符串排序**，因此

```jsx
let arr = [ 1, 2, 15 ];
arr.sort();
alert( arr );  // 1, 15, 2
```

sort接受一个参数来控制对比较的定义，当前一个值大于后一个值时，返回值大于0，类推。

```jsx
function compare(a, b) {
  if (a > b) return 1; // 如果第一个值比第二个值大
  if (a == b) return 0; // 如果两个值相等
  if (a < b) return -1; // 如果第一个值比第二个值小
}
```

## reverse【就地】

就地对数组进行逆转操作，并且会返回修改后的数组。

## reduce, reduceRight

reduce用于将数组归约为单个值，其回调函数参数分别为为accumulator, x, i, a，返回值作为下一个元素调用时的accumulator值。accum默认值是initial（reduce的第二个参数），如果没有提供，就是数组的第一个元素；如果数组为空而没有第一个元素，reduce会直接报错。

reduceRight与reduce的遍历方向相反。

## 数组方法的thisArg

find, filter, map方法都支持在最后传入一个thisArg，用于替换其回调函数的this。

## fill

fill(value, start?, end?)将数组用value从start填充到end，如果范围超越了数组的长度，不会额外创建位置。如果start和end均不填，则填充整个数组，如果end不填，则从start填充到数组的末尾。
