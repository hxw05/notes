# 22 原型

每一个对象上都有一个隐藏的属性[[Prototype]]，可以通过设置对象的__proto__属性来设置它。当一个属性或方法在对象上找不到时，就会顺着__proto__的指向逐级寻找。

__proto__可以指向一个对象或者null，如果是其他类型会被忽略。这样的行为类似于继承。限制条件：

- __proto__不能形成闭环。

__proto__是一个因为历史原因留下来的对[[Prototype]]的getter/setter，现代的方式是使用Object.get/setPrototypeOf方法去设置Prototype。

__proto__存在于Object.prototype上面。

## 原型只用于读操作

对一个对象的写/删除操作不会应用在原型上。

```jsx
let hamster = {
  stomach: [],

  eat(food) {
    this.stomach.push(food);
  }
};

let speedy = {
  __proto__: hamster
};

let lazy = {
  __proto__: hamster
};

speedy.eat("apple");
alert( speedy.stomach ); // apple
alert( lazy.stomach ); // apple
```

一个直观的修复方法是给speedy和lazy都加上stomach，来避免原型链的搜索。

另一种方法是，eat函数内部由于是对this.stomach.push的调用，因此相当于是先读取了this.stomach。this.stomach.push会导致原型链的搜索，从而搜索到hamster上的stomach并对其执行push操作。如果将this.stomach.push(food)修改为赋值形式this.stomach=[food]，则由于是写操作，不会进行搜索，这样即使一开始没有stomach属性也会被加上。

## this的绑定不受原型影响

调用obj的方法中的this永远是obj，而不是原型链上的某个对象，即使调用的方法来自于那个对象。

```jsx
let animal = {
  walk() {
    if (!this.isSleeping) {
      alert(`I walk`);
    }
  },
  sleep() {
    this.isSleeping = true;
  }
};

let rabbit = {
	__proto__: animal,
}

rabbit.sleep(); // 方法来自animal，但效果在rabbit上，因为this是rabbit
animal.walk(); // 无效，animal.isSleeping是undefined
```

## for … in会包含原型链上的属性

如果希望获得专属于当前对象的属性，则可以用hasOwnProperty方法（存在于object实例上）判断某个键是否为当前对象本身的属性。

- hasOwnProperty是Object.prototype上设置的方法，因此对于object实例来说它也是继承来的，但不会出现在for … in里面。因为，通过Object.getOwnPropertyDescriptors(Object)可以看到，其上面所有的静态方法的enumerable都被设置为false。

## F.prototype

每一个函数对象都有一个名为prototype的一般属性，其默认值为{constructor: 函数本身}

new操作符在创建对象时，除了会完成所有this.xxx的赋值，还会将新的对象的__proto__指向F.prototype。因此，我们可以通过修改prototype的值来实现继承，这样new出来的对象将有正确的__proto__指向。

- 建议使用F.prototype.xxx=的形式进行赋值，避免覆盖掉原始的constructor属性
- 注意F.prototype是一个对象，也就是一个引用。__proto__也是一个引用。对该引用的修改也会反映在后面的运行上。

```jsx
function Rabbit() {}
Rabbit.prototype = {
  eats: true
};

let rabbit = new Rabbit();

Rabbit.prototype = {}; // {}是一个新的对象

alert( rabbit.eats ); // true
// 这是因为rabbit.eats指向的仍然为修改之前的原对象，
// 只不过这个对象现在只被rabbit.__proto__指向
```

```jsx
function Rabbit() {}
Rabbit.prototype = {
  eats: true
};

let rabbit = new Rabbit();

Rabbit.prototype.eats = false;

alert( rabbit.eats ); // false
// Rabbit.prototype与rabbit.__proto__指向同一个对象，
// 对prototype的修改也会反映在__proto__上
```

## 原生prototype

JS的内部实现中，原生的对象都使用到了prototype，例如将一个object表示为[object Object]的toString方法就存在于Object.prototype上。

所有原生对象最终都指向Object.prototype，Object.__proto__则指向null。

prototype是可以修改的，通常用于实现polyfill。

## Object.create(proto, descriptors?)

Object.create函数可以创建一个以proto为原型的object。

这个时候可以做一个小试验：所有的object默认都以Object为其原型。是否能够创建一个不以Object为原型的object，就像Object本身那样？可以，将proto设置为null，就可以创建一个原型链顶层的object。

```jsx
String(Object.create(null))
// Uncaught TypeError: Cannot convert object to primitive value
// 因为这个新的对象不再自动具有toString方法了
```

这个没有原型的对象还具有一个特点：它的__proto__由于没有从Object继承来，可以作为一般的属性使用。这一对象被称为pure dictionary/very plain对象。

使用以下方法可以创建一个真正完整的对象拷贝（但不是深拷贝）

```jsx
let clone = Object.create(
  Object.getPrototypeOf(obj),
  Object.getOwnPropertyDescriptors(obj)
);
```
