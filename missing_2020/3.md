## vim介绍

Vim is a modal editor: has multiple operating modes

在编程过程中可能会有不同类型的需求，例如阅读代码、做一些小的修改或者写一些大的功能。


- normal mode
    
    all the different key combination behaves in one way
    
    navigate around a file, reading things, going from file to file…
    
    - 按下的按键会用于进行操作或导航
    - 按下特定按键，进入特定模式，按下`ESC`返回normal mode
    
    > …a lot of programmers rebind one of the keys on their keyboard to be escape because it’s really inconvenient to reach up with your left pinkie to press that tiny little `Escape` key in the corner of your keyboard. (8:50)
    > 
    
    For key rebinding see lecture notes
    
- insert mode `I`
    
    type in text
    
- replace mode  `R`
- selection modes
    - visual mode `V`
        - visual line `S-V`
        - visual block `C-V`
- command-line mode `;`(colon)

## 记号

- 键盘上可以直接按下的按键，就直接用它的名字来表示，如`i`键
- 按键组合，以Control+V为例，三种记法
    - `^V`(caret v)
    - `Ctrl-V`
    - `<C-V>`

## 基本使用

For vim configuration see lecture notes

- 左下角显示的是当前所处的模式，normal mode不显示
- 在vim中的功能都可以脱离鼠标（只用键盘来完成），具体地，是通过前面所列的那些mode来完成(12:29) in particular through the command-line mode

一些可以在command-line mode里面用的指令

- 退出 `:quit` `:q`
- 写入（保存）`:w`
- 帮助 `:help 指令`
    - eg. `:help :w`
    - 注意`:w`和`w`是两种东西，前者代表指令，后者是normal mode的功能按键

## buffers vs. windows vs. tabs

- vim维护着一组open buffers，在这里用open buffers来代指一般意义上“打开的文件”
- 可以有若干个tab，tab内可以有window
    - window可以理解为编辑器内分屏
- buffer和window并非一对一的关系，可以用两个window开同一个buffer

`:q`实际上指的是关掉当前的window，如果没有window可关就是关闭vim

`:qa`关闭所有window，等同于关闭vim

## what is normal mode for?

fundamentally interesting idea: the interface is a programming language.

vim界面及其交互设计本身就是一种编程语言，通过类似于高级语言中的组合，可以完成很多复杂的工作，极大提高效率。

## Normal Mode 指令

Note: 下面的符号均代表键盘上的按键

### hjkl

减少使用键盘上的方向键，用hjkl来代替，提高效率。

←h↑j↓k→l

### wbe

w前进，b回退（**以单词为单位**），e前往单词的结尾

### 0$^

0去到当前行的开头，$去到当前行的末尾

^去到当前行第一个非空字符（适用于eg. 有缩进的行）

### ^U, ^D

相当于jk的加速

### G, gg

G到达末尾（end），g到达开头（home）

G=`Shift`+`G`

See lecture note

这些指令有些具有助记符特性，采用相应含义单词的开头来代表，0$^这种可能受到了正则表达式的启发。

### HML

将指针移动到当前显示内容的顶部（H）、中部（M）、底部（L）

### Ff, Tt

F stands for “find” and T stands for “to”

Ff+字符，将指针移动到当前行、指针前（F）/后（f）的第一个指定字符的位置。

Tt+字符，将指针移动到当前行、指针前（T）/后（t）的第一个指定字符之后（T）/前（t）的位置。

### Oo

在指针所在行的上（O）/下一行（o）创建一个新的空行，并进入INSERT模式

### u, ^R

Undo

Redo

> 提问（30:10）
> 
> Q: Undo撤销的范围是什么？
> 
> A: Normal Mode下的一次操作、一整个Insert Mode下的编辑
> 

### d, c

d stands for delete, c stands for change

dw删除指针当前所在的单词

de从指针当前位置删除到单词末尾

c与d的表现大致相同，c会自动进入INSERT模式

双d（dd）、双c（cc）作用于指针所在的一整行

### x

删除指针所在的单个字符。如果是caret（竖线光标），则表现为删除光标**之后**的字符。

### r

按下r后，vim会等待输入一个字符，并将光标所在处替换为输入的字符。

### y, p

y stands for *yank* (the terminology vim used for copying)

y复制，p粘贴

yw复制当前单词

yy复制当前行

### weird commands… like `~`

~用于翻转选区字母的大小写

%在一对符号之间往返 (moving back and forth between matching parentheses）

/用于搜索（`n`可用于跳转到下一个匹配项）

.用于重复先前的修改

### counts（次数）

先输入数字，再输入指令可用于批量操作，例如

- 向下移动4行，可以是jjjj或者4j
- visual mode下，e表示向后选择一个单词，3e表示向后选择三个单词
- 7dw - 向后删除7个单词

### modifiers（修饰符）

- a - around
- i - inside

以markdown一个链接为例

```markdown
[example link](example.com)
```

如果需要更改方括号内的内容

- 一种方法是将指针放在方括号开头，按下2cw，或者2dw+INSERT来实现
- **使用修饰符：**将指针放在包围的范围内，键入`ci[`表示change inside [，也就是修改[]包围起来的所有内容
- 类似地，`di(`表示delete inside (，也就是删除()包围起来的所有内容
- `da(`表示delete around (，除了删除()包围起来的内容以外，还会删掉括号

## Visual Mode

Normal Visual Mode - 按v进入，主要用于进行选择，可以使用hjkl来调整选区

Line Visual Mode - 按V进入，用于行选择，可使用jk来调整选区（上下）。

Block Visual Mode - ^V进入，用于选取矩形选区

![image.png](3%20Editors%20(vim)/image.png)

### y

选择一个选区以后，按下y复制该选区，并退回到normal mode

### e

向后以单词为单位进行选择

可以在前面加上数字，例如3e向后选择三个单词
