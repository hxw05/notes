# 2. Shell 工具及脚本

## 脚本基础

### When defining variables, space is important

- `foo=bar; echo $foo`
- `foo = bar` -> command not found: foo

这意味着空格是用来分隔参数的，不能多也不能少

### 字符串

类似于PHP语言，字符串既可以用单引号又可以用双引号包围，其中只有双引号会替换其中的变量。

### source

```bash
# mcd.sh
mcd () {
	mkdir -p "$1"
	cd "$1"
}
```

将mcd载入到当前会话中：`source mcd.sh`

### 常用特殊记号

| 符号/操作符 | 名称/描述 | 详细说明与示例 |
| :--- | :--- | :--- |
| `$0` | 脚本名称 | 调用脚本时指定的名称。<br>• `./a.sh` 调用，值为 `./a.sh`<br>• `/path/to/a.sh` 调用，值为 `/path/to/a.sh` |
| `$1` ~ `$9` | 位置参数 | 脚本接收到的第 1 到第 9 个参数。 |
| `$?` | 错误码 | 前一个指令的退出状态。<br>• `0` 代表没有错误。<br>• 非 `0` 代表发生了错误。 |
| `$_` | 最后参数 | 前一个指令的最后一个参数。<br>• 例：执行 `mkdir test` 后，`cd $_` 等同于 `cd test`。 |
| `$#` | 参数个数 | 传递给脚本的参数数量（从 `$1` 开始计数）。 |
| `$$` | 进程 ID | 当前 Shell 脚本或进程的 PID。 |
| `$@` | 所有参数 | 引用传递给脚本的所有参数，通常用空格分隔。 |
| `!!` | 上一条指令 | 执行上一条指令（就地执行，不同于按 ↑ 键）。<br>• 例：`sudo !!` 会以 sudo 权限重新执行上一条命令。 |
| `\|\|` | 逻辑或 | `a \|\| b`：如果 `a` 执行**失败**（返回非0），则执行 `b`。 |
| `&&` | 逻辑与 | `a && b`：如果 `a` 执行**成功**（返回0），则执行 `b`。 |
| `$()` | 命令替换 | 将命令的执行结果赋值给变量或嵌入字符串。<br>• 例：`foo=$(pwd)` |
| `>/dev/null` | 忽略标准输出 | 将 STDOUT 重定向到空设备，即忽略正常输出。 |
| `2>/dev/null` | 忽略错误输出 | 将 STDERR 重定向到空设备，即忽略错误信息。<br>• 注意：这不会改变错误码（`$?`）。 |

```bash
echo 123
echo $?
# => 0
grep foobar mcd.sh
echo $?
# => 1
true
echo $?
# => 0
false
echo $?
# => 1
```

```bash
nothing
# zsh: command not found: nothing
nothing 2>/dev/null
# 无输出
nothing 2>/dev/null;echo $?
# 127
echo 2 > /dev/null
# 无输出
echo 2> /dev/null
# 空输出（2被看做和>一体，定向STDERR）
```

### test指令

`test`提供了许多用来检测的操作

| 操作符 | 描述 |
| :--- | :--- |
| `-b file` | 如果文件存在且是块特殊文件，则为真。 |
| `-c file` | 如果文件存在且是字符特殊文件，则为真。 |
| `-d file` | 如果文件存在且是目录，则为真。 |
| `-e file` | 如果文件存在（无论何种类型），则为真。 |
| `-f file` | 如果文件存在且是普通文件，则为真。 |

## 通配符写法/文件名展开

用通配符来指代文件称为globbing，对应的文件正则（暂且称为）称为glob pattern

|通配符|含义|
|:--|:--|
|`*`|展开为若干个字符（0+）|
|`?`|展开为单一字符|
|`{x,y,...}`|用于同时指代多个并列项*|
|`{a..z}`|具有语义的并列项，其中a..z是这两个字母所构成的字母表范围|

例如
- `convert image.png image.jpg`  → `convert image.{png,jpg}`
- `touch project{1,2}/src/test/test{1,2,3}.py` → 会转变为6个对象，实质上是这些并列项构造的集合的笛卡尔积

## `diff`

用重定向和diff比较目录中文件上的不同：`diff <(ls a) <(ls b)`

## Shebang

最开头的注释是shebang，用来指示该脚本默认的运行方式，例如py文件的shebang应该指定为类似于`/usr/local/bin/python`的解释器。

```bash
#!/usr/local/bin/python
```

或者可以用到`env`来获取到对应解释器名称在环境变量中指定的位置

```bash
#!/usr/bin/env python
```

这样做的好处是让程序在不同机器上的兼容性提高，避开了直接绝对指定解释器路径。

## 定位文件

### `find`

```bash
find . -name src -type d
find . -path '**/test/*.py' -type f
```

- type指定搜索目标的类型
- name指定搜索目标的名称（可以glob）
- path搜索目标预定路径（可以glob）

```bash
find . -mtime -1 #查找前一天以内修改过的文件
```

- mtime（modification time）指定修改时间范围，-1表示前一天

```bash
find . -name "*.tmp" -exec rm {} \; # 删除所有此类文件
```

### `fd`（比`find`简单且更快）

直接传入带正则表达式的字符串即可。

```bash
fd "*.py"
```

### `locate`

locate用于全盘搜索文件名，本质是在本地带索引数据库中搜索传入的**文件名**（作为substring）

```bash
locate missing-semester
```

在使用之前需要先建库，如果没有建库在macos上会出现提示如下

```
WARNING: The locate database (/var/db/locate.database) does not exist.
To create the database, run the following command:

  sudo launchctl load -w /System/Library/LaunchDaemons/com.apple.locate.plist

Please be aware that the database can take some time to generate; once
the database has been created, this message will no longer appear.
```

然而给到的这个指令并不能完成建库，不知道为什么：https://superuser.com/questions/1681474/locate-command-does-not-work-on-mac-terminal。真正能完成建库的指令是

```bash
sudo /usr/libexec/locate.updatedb
```

大概花费一分钟左右时间完成。

## 搜索文件内容

### `grep`

grep用于搜索文件内容。

```bash
grep foobar mcd.sh # 在mcd.sh里面搜索
grep -R foobar . # 当前目录下递归搜索并列出
```

### `rg`

快速进行全盘文件内容搜索。

如果你记得你写了一些代码但是不知道放在哪里了就可以用这个。

```bash
rg "import requests" -t py -C 10 ~/scratch
```

- `-t`是文件类型
- `-C`是需要显示的前后行数

```bash
rg -u --files-without-match "^#\!" -t sh # 寻找不包含shebang的所有sh文件，包括已隐藏的
```

- `-u`表示包含已隐藏的文件
- `—file-without-match`返回不满足pattern的文件
- `—stats` 显示一些详细的统计信息

### `fzf`（非自带）

用于交互式地查询输入内容中的字符串
