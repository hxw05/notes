### 各列含义

第一列是文件的模式（file mode）。

第1位表示文档类型（entry type）
        
| 第一位 | 文件类型 |
| --- | --- |
| - | 普通文件 |
| b | block special file，块特殊文件 |
| c | character special file，字符特殊文件 |
| d | 目录 |
| l | 符号链接 |
| p | 管道 |
| s | 套接字 |
| w | Whiteout |

后9位每三位为一组，依次对应三种身份所拥有的权限，身份顺序为owner（文件所有者）、group（文件所有者所在的群组）、others（其他用户），每一组的三位所代表的权限顺序为：readable、writable、executable
        

| 位数 | 可能出现的权限字符 | 含义 | 备注 |
| --- | --- | --- | --- |
| 1 | r | 可读 |  |
|  | - | 不可读 |  |
| 2 | w | 可写 |  |
|  | - | 不可写 |  |
| 3 | S | 不可执行或不可搜索且设置了setuid mode或者setgid mode |  |
|  | s | 可执行或可搜索且设置了setuid mode或者setgid mode |  |
|  | x | 可执行或可搜索 |  |
|  | - | 不可读、不可写、不可执行、不可搜索、非setuid或setgid、非黏性 |  |
|  | T | 不可执行或搜索，黏性 | 只会出现在others段 |
|  | t | 可执行或搜索，黏性 | 只会出现在others段 |

对于目录，r权限表示是否可以查看该目录底下有哪些内容。w权限表示是否可以重命名、删除、创建目录下的文件。x权限表示是否可以**搜索（search）**该目录，更确切地说，是否可以进入该目录。

- eg. 假设目录A下有文件a，你具有wa但不具有wA权限，那么你只能编辑其内容，却无法将其删除。
- eg. 如果你想要进入 /a/b/c/d（或访问其中的文件），则必须同时具有xa、xb、xc、xd权限。

对于第3位，这里列出的权限字符同一个文件或目录可能拥有多个，而结果中只会显示第一个（按照表中从上往下顺序）。eg. 同时具有Ssx，第3位只会显示S。

除了常见的rwx，之外还有s（S）、t（T）两种权限，对应了setuid、setgid和sticky bit三种概念，参考https://tonydeng.github.io/2007/04/06/the-difference-between-setuid-setgid-and-sticky-bit-in-linux/。

- setuid、setgid
    - 具有该权限（s）的文件，在执行时自动取得文件所有者（owner）的权限，而非沿用执行者的权限
    - 具有该权限的目录，其内部创建的文件自动继承父级目录的组，而非沿用创建者的组
- sticky
    - When a directory has the sticky bit set, its files can be deleted or renamed only by the file owner, directory owner and the root user.

## Rewiring/redirection

在终端中有两个基本流，即输入和输出流，它们默认情况下都是终端本身，这解释了 `echo abc` 会直接在终端上输出abc的问题。

- `<`  - rewire the input stream of the program to be the contents of the file
- `>`  - rewire the output of the program into the file

eg.

1. `cat abc.txt` 显示abc.txt的内容
2. `cat < abc.txt` 也是显示abc.txt的内容，但是步骤不同

a中abc.txt作为cat的参数传入，由cat读取文件内容并展示出来

b中bash先将abc.txt读取，然后传入作为cat的输入，cat将输入它的内容展示出来

不使用cp指令复制文件：`cat < xxx > xxx2` （第一个`<`可以不带）。注意这里的重定向对于程序而言是透明的。

- `|`  - take the output of the program to the left and make it the input of the program to the right.

## `tail`

## `sudo`, `tee`: Changing contents in the kernel

Linux系统中的内核存储在专门的目录中，通过修改这一目录中文件的内容可以达成很多有趣的事情，例如当收到邮件以后，就让键盘上的ScrollLock键亮起来（假设该键已经被弃用）（44:45）。但是要修改的前提是需要有root权限。

一般而言，sudo后跟指令就可以以超级管理员身份执行该指令。但是，需要特别注意重定向符号并不是在sudo的范围之内。

eg. 修改kernel中笔记本屏幕亮度参数文件intel_backlight/brightness
    
- `echo 500 > brightness`  → permission denied
- `sudo echo 500 > brightness`  → *permission denied*

这是因为sudo echo 500是一个部分，重定向又是另外的一个步骤，即此处打开brightness的操作是由bash（按照当前用户权限）完成而非sudo，又一次强调了前文所说的rewiring/redirection与程序无关。

正确的做法（非sudo用户）是使用sudo来运行tee，并用管道符将输入的内容传递给tee（接收输入，并将输入的内容写入到文件里）。

`echo 500 > sudo tee brightness`
    

终端提示符中的美元符号`$`表示当前运行的用户并非root，root用户用`#`表示（40:45）。
